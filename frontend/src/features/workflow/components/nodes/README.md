# 工作流节点学术手绘本风格系统 (Academic Paper UI Styling System)

在 StudySolo 的工作流引擎中，我们为流程中的不同计算和业务环节设计了一套统一的 **“学术稿件 / 手绘本”** 卡片设计方案。

这套体系能够将高度技术化和抽象的数据流图，转换为仿佛教授/学生的书桌稿纸般的治愈、沉静的外观，大幅缓解使用 AI 工具的疲劳感。

我们预设了 **10种独立的质感主题**，可以在拓展新类型节点时开箱即用：

## 1. RAW_DATA (数据源/输入)
- **色调**: Slate (冷灰板岩)
- **造型特点**: 细虚线外边框，无内部修饰。
- **隐喻**: 撕下来的草稿纸或是打孔手账纸，表示未经处理的原始输入。
- **关联类型**: `trigger_input`

## 2. ANALYSIS (逻辑分析)
- **色调**: Emerald (深翡翠绿)
- **造型特点**: 粗实线外边框 + 虚线内嵌内框。
- **隐喻**: 教材上的严谨分析页，双框代表了深刻的推理与解构。
- **关联类型**: `ai_analyzer`, `content_extract`, `compare`

## 3. GENERATION (内容生成)
- **色调**: Indigo (深邃靛蓝)
- **造型特点**: 极粗外框 + 实线内细框。
- **隐喻**: 厚重的钢笔书写/正文生成，代表创造性的核心工作区域。
- **关联类型**: `outline_gen`, `summary`

## 4. FINAL_REPORT (正式报告/终稿)
- **色调**: Slate Blue/Navy (藏青蓝)
- **造型特点**: 边缘自带虚线间隙（内嵌缝合线）并辅以弥散外发光圈 (`ring`)。
- **隐喻**: 最终装订成册的论文报告页。
- **关联类型**: `chat_response`, `merge_polish`

## 5. EXTERNAL_TOOL (外部检索)
- **色调**: Cyan (青琉璃)
- **造型特点**: 仅有极粗的**左侧边框**，辅以极细的其他三边，无内边。
- **隐喻**: 贴在书页旁边的胶带便签/侧边书签，代表外部抓取和引用动作。
- **关联类型**: `knowledge_base`, `web_search`

## 6. ACTION_IO (系统读写)
- **色调**: Zinc (工业锌灰)
- **造型特点**: 细密集点线（Dotted）外框 + 单点半透明内框。
- **隐喻**: 自动化打字机输出或系统操作指令单。
- **关联类型**: `write_db`, `export_file`

## 7. CONTROL_FLOW (逻辑控制)
- **色调**: Amber (琥珀橙)
- **造型特点**: 标准双实线带半透明虚线。
- **隐喻**: 施工/跳转/警告标记，用于控制与引导后续步骤方向。
- **关联类型**: `logic_switch`, `loop_map`

## 8. VISUALIZE (图表渲染)
- **色调**: Fuchsia (高亮紫)
- **造型特点**: 特别加入了内嵌阴影发光 (`shadow-[inset_0_0_0_2px...]`) + 小边距实线内框。
- **隐喻**: 悬挂在主页上的相框或是重点数据展示板。
- **关联类型**: `mind_map`

## 9. ASSESSMENT (考核测试)
- **色调**: Rose (玫瑰红)
- **造型特点**: 上下方向带截断的内边框。
- **隐喻**: 考卷/错题本中老师的红笔批改与划线隔离区。
- **关联类型**: `quiz_gen`, `flashcard`

## 10. FEEDBACK (评估反馈)
- **色调**: Teal (青黑笔迹)
- **造型特点**: 粗实线包边加虚线点阵内修饰。
- **隐喻**: 导师的点评便利贴。
- **关联类型**: *(作为扩展的 Fallback 主题预留)*

---

### 如何在代码中使用
所有主题控制逻辑均收敛于 `frontend/src/features/workflow/components/nodes/AIStepNode.tsx` 中的 `getNodeTheme` 方法：
- CSS 基于 Tailwind，支持 `.dark` 夜间模式优雅回退（白天是深色边框白纸，夜晚是浅色边框黑纸）。
- 如果要给新 `nodeType` 增加特定样式，只需进数组加入其名称即可。如果没有匹配，默认回退至第 10 套反馈风格。
