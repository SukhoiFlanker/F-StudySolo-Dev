### #16 修复：私有画布页 401 Token 过期导致 notFound

问题：访问 `/c/{workflowId}` 时，若 Supabase access_token 已过期，服务端 `fetchWorkflowContentForServer` 直接返回 `null`，页面显示 404 而非刷新 token 重试。

根因：客户端 `authedFetch` 有 401 自动刷新重试逻辑，但服务端渲染路径（Server Component → workflow.server.service → workflow.service）缺少对应机制。过期 token 发到后端被中间件拒绝，返回 401，前端静默吞掉错误返回 `null`。

修复：
- `workflow.service.ts`：`fetchWorkflowContent` 在收到 401 时抛出 `ApiError` 而非静默返回 `null`
- `workflow.server.service.ts`：新增 `tryServerRefresh()` 函数，使用 `refresh_token` cookie 调用后端 `/api/auth/refresh` 获取新 token；`fetchWorkflowContentForServer` 捕获 401 后自动刷新并重试一次
