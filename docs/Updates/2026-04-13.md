# 2026-04-13 更新日志

## 1. Phase 5 第三波：Agent Gateway 核心实现 (S3.2 + S3.3 + S3.4)

### 完成内容

**S3.2 agents.yaml + registry**

- 创建 `backend/config/agents.yaml`，注册 `code-review-agent`（url: 127.0.0.1:8001, timeout: 45s）
- 实现 `AgentRegistry`：启动时从 YAML 加载，支持 `get / list_all / list_enabled`
- 实现 `AgentMeta` 数据模型（name / url / timeout / max_retries / api_key_env / models / enabled）

**S3.3 agent_gateway 核心实现**

- `AgentCaller`：non-stream（httpx POST → JSON 解析）+ SSE 透传（httpx stream → 逐 chunk yield）
- `HealthChecker`：探测 `/health/ready`，30s TTL 缓存，支持 invalidate
- `AgentGateway` 主类：
  - `discover()`：返回已注册且健康的 Agent 列表
  - `call()`：non-stream 调用，含 404/503 前置检查 + 审计日志
  - `call_stream()`：SSE 流式调用，审计日志在流结束后记录
  - `_build_headers()`：从 `os.environ` 解析 `api_key_env` → `Authorization: Bearer <key>`
  - 审计日志：stdout JSON 格式（event / request_id / agent / user_id / stream / status / duration_ms）

**S3.4 /api/agents/* 路由**

- `GET /api/agents`：列出所有可用 Agent（需 JWT）
- `POST /api/agents/{name}/chat`：调用指定 Agent（支持 stream / non-stream，需 JWT）
- `GET /api/agents/{name}/health`：查询指定 Agent 健康状态（需 JWT）
- 已注册到 `router.py`，路径 `/api/agents/*`
