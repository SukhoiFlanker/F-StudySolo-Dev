### #01 运行记忆页面画布化升级（/m/[id]）

将 `/m/[id]` 运行记忆页面从纯卡片列表视图升级为与 `/s/[id]` 一致的全屏画布视图，完整展示工作流执行过程。

**后端**
- `workflow_runs.py`：`get_run_detail` 和 `_build_public_run_response` 扩展查询 `nodes_json, edges_json`，一次请求返回完整画布数据 + 执行记录

**前端类型**
- `types/memory.ts`：`WorkflowRunDetail` 新增可选字段 `nodes_json` 和 `edges_json`

**前端路由 m/[id]**
- `layout.tsx`：从文档流布局重写为全屏画布布局（参考 `/s/[id]/layout.tsx`），浮动顶栏
- `MemoryView.tsx`：全面重写为画布 + 浮动面板模式
  - 复用 `ReadOnlyCanvas` 渲染完整工作流画布
  - `inject-traces.ts` 纯函数将 trace 执行数据注入节点 data，通过 `AIStepNode` → `NodeResultSlip` 展示每个节点的输入/输出/状态/耗时
  - 左侧浮动面板：运行摘要（时间/耗时/Token/状态/节点统计/用户输入）
  - 右侧浮动按钮：进入编辑 + 分享开关 + 复制链接
  - 孤立 trace 处理：画布中已删除的节点的执行记录在底部浮动面板展示
  - Fallback：workflow 已删除时降级为卡片列表视图
- `inject-traces.ts`：新建，trace → node data 映射 + 孤立 trace 识别

**组件复用**
- 直接复用 `ReadOnlyCanvas`、`AIStepNode`、`NodeResultSlip`、所有节点渲染器、边组件，零修改

### #02 运行记忆页面：分享确认弹窗 + 展开全部详情

**分享确认机制**
- 点击"分享"按钮不再直接切换状态，而是弹出确认对话框
- 对话框说明分享意味着什么：画布结构、节点输入输出、执行详情、用户输入文本均会公开
- 用户确认后才执行分享操作；取消分享（已公开→取消）无需确认，直接执行

**展开全部详情**
- 分享按钮下方新增"展开详情/收起详情"按钮
- 点击后通过 `CustomEvent('workflow:toggle-all-slips')` 广播，所有画布上的 `NodeResultSlip` 同步展开或收起
- `NodeResultSlip.tsx` 新增事件监听器，响应全局展开/收起指令
