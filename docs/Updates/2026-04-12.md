## 1. Phase 4B：repo-aware 同序风格等价 identifier canonicalization 落地

今天继续沿着 Phase 4B 做一个 repo-aware 利用治理闭环：只收口 `code-review-agent` 的 shared identifier canonicalization 与连续窗口命中逻辑，不触碰 Gateway、不改公共 API、不引入本地仓库读取。

### 完成内容
1. **新增 repo-aware canonical identifier helper**

   - 新增：
     - `canonical_shared_identifier(...)`
     - `repo_aware_identifiers(...)`
   - 统一把 camelCase / PascalCase / snake_case / acronym 风格收口为 `lowercase_with_underscores`
   - 命中示例：
     - `requestBody -> request_body`
     - `HTTPClient -> http_client`
     - `apiURL -> api_url`
     - `userID -> user_id`
     - `errorCount -> error_count`
2. **shared identifier overlap 统一改走 canonical form**
   - `_shared_identifier_candidates(...)` 改为基于 canonical overlap 求交
   - generic token 过滤继续保留，但只过滤单词级 token
   - `request_body`、`status_code`、`error_count` 这类 compound token 保留
3. **连续窗口评分改为复用同一套 repo-aware identifier 真相**
   - `agents/code-review-agent/src/core/agent.py`
   - `_preview_forwarded_context(...)` 按行命中统计改为复用 `repo_aware_identifiers(...)`
   - 连续窗口评分公式不变，仍按：
     - 唯一 shared identifier 数
     - shared identifier 命中总次数
     - 命中的行数
     - 更早的窗口起点
4. **锁住顺序边界，不放宽到子词重排**
   - `cacheAdapter` 与 `adapter_cache`
   - `userID` 与 `id_user`
   - 以上都继续视为不重叠
5. **repo-aware prompt hints 值语义已统一**
   - 例如：
     - `renderbadge -> render_badge`
     - `totalcount -> total_count`
     - `formatcount -> format_count`
     - `handleerror -> handle_error`

## 2. Phase 4B：unknown-rule groundedness 同序风格等价收口

继续沿着 Phase 4B 做上游治理细化，这一轮不再扩 repo-aware overlap 本身，而是把 unknown live upstream 的 `Title` / `Fix` groundedness 判断也收口到同一套 style-equivalent canonical identifier 真相，避免 camelCase 与 snake_case 仅因命名风格不同而被误重写为占位文案。

### 完成内容
1. **unknown-rule groundedness reference identifiers 改走 canonical form**
   - `agents/code-review-agent/src/core/agent.py`
   - `collect_live_upstream_reference_identifiers(...)`
   - 从原始 `extract_identifiers(...)` 改为复用：
     - `upstream_review.repo_aware_identifiers(...)`
2. **substantive governed identifier 判断改走 canonical form**
   - `extract_substantive_governed_identifiers(...)`
   - 对 title / fix 中的 identifier 先 canonicalize，再做 groundedness 判断
   - 允许：
     - `requestBody -> request_body`
     - `statusCode -> status_code`
     - `HTTPClient -> http_client`
     - `apiURL -> api_url`
     - `userID -> user_id`
3. **继续锁住顺序边界**
   - 仍然不放宽到子词重排
   - `cacheAdapter` 与 `adapter_cache`
   - `userID` 与 `id_user`
   - 以上继续视为 foreign identifier
4. **展示文本保持原样**
   - canonicalization 只用于 groundedness 比较
   - 不会把最终 `Title` / `Fix` 的自然语言展示文本强行改成 underscore 形式
5. **其他治理链路保持不变**
   - 不改 `evidence_is_anchored_to_review_target(...)`
   - 不改 unknown `severity -> medium`
   - 不改 `rule_id` 治理

## 3. Phase 4B：known-rule ID 同序风格等价收口

继续沿着 Phase 4B 做 live upstream 治理细化，这一轮不再扩 unknown-rule groundedness，而是把 known rule 的 `rule_id` 命中也收口到同序风格等价 canonicalization，修复 camelCase / PascalCase / kebab-case 只因命名风格不同而误落入 unknown-rule 治理分支的问题。

### 完成内容
1. **新增 known-rule 专用 canonical helper**
   - `agents/code-review-agent/src/core/agent.py`
   - 新增：
     - `canonical_known_rule_id(...)`
     - `resolve_known_live_rule_spec(...)`
   - 规则固定为：
     - `strip()`
     - 只允许 `[A-Za-z0-9_-]`
     - 仅按 `_` / `-` 切段
     - 每段再按 camel / Pascal / acronym 边界切分
     - 子词转小写并用 `_` 连接
     - 保留顺序，不做重排
2. **live upstream known-rule lookup 改走 canonical resolution**
   - `normalize_live_upstream_findings(...)`
   - 不再只做 `KNOWN_RULE_SPECS.get(finding.rule_id)` 精确匹配
   - 现在会先走 `resolve_known_live_rule_spec(...)`
   - 命中后统一收口到本地规则表里的：
     - `rule_id`
     - `title`
     - `severity`
     - `fix`
3. **style-equivalent known rule ID 现在会稳定命中**
   - 例如：
     - `debugArtifact -> debug_artifact`
     - `debug-artifact -> debug_artifact`
     - `hardcodedSecret -> hardcoded_secret`
     - `tlsVerificationDisabled -> tls_verification_disabled`
     - `broadExceptionSwallow -> broad_exception_swallow`
4. **继续锁住误报边界**
   - 不做子词重排
   - `artifact_debug` 不会命中 `debug_artifact`
   - `disabled_tls_verification` 不会命中 `tls_verification_disabled`
   - `debug/artifact` 这类非法分隔符不会参与 known-rule 命中，仍走原有 unknown-rule 安全治理
5. **去重语义同步受益**
   - 若 upstream 同时返回 `debug_artifact` 与 `debugArtifact`
   - canonical known-rule 命中发生在 display key 去重前
   - 最终只保留一条 canonical finding
