### 【面向开发者】

#### v0.1.001 🏗️ 技术栈确立与深度研究

经过严格版本兼容性验证，最终确立双语言技术栈：

**前端（`frontend/`·pnpm 管理）**
- Next.js `16.1` · React `19.2` · TypeScript `5.x`
- Tailwind CSS `4.1`（Oxide 引擎） · Shadcn/UI
- @xyflow/react `12.10.0`（工作流画布） · Framer Motion `12.x`
- Zustand `5.x`（状态管理）

**后端（`backend/`·pip/uv 管理）**
- Python `3.11+` + FastAPI `0.115+`（替代原定 Node.js Express）
- Gunicorn `23+` + Uvicorn `0.34+`（ASGI WSGI 双层架构）
- Pydantic `2.10+` · supabase-py `2.12+` · openai SDK `1.60+`
- sse-starlette `2.2+`（AI 流式输出）

**选型核心理由（FastAPI 相比 Node.js 的优势）**
- Pydantic V2 自动请求/响应验证，Vibe Coding 生成代码质量极高
- 异步原生 ASGI，I/O 密集型场景（AI API代理）性能最优
- 自动生成 Swagger UI 文档，前后端 AI 联调效率翻倍
- 与 AI 大模型技术栈天然贴合，学习价值极高

**数据库 & 云服务**
- Supabase **Pro** $25/月（永不暂停 · 8GB DB · pgvector · 每日备份）
- 火山引擎 doubao-2.0-pro（简单任务 · 200W Token/日免费）
- 阿里云百炼 qwen3-turbo（复杂任务）
- 双模型均兼容 OpenAI API 格式，一个 SDK 统一调用

**关键配置参数**
- 前端端口：`2037`
- 后端端口：`2038`
- ECS 规格：2核4G · Alibaba Cloud Linux 3

#### v0.1.002 📁 项目文档体系建立

建立规范化的 `docs/` 目录结构：

```
docs/
├── README.md             # 文档总览
├── updates/              # 更新日志（本目录）
│   ├── README.md         # 更新日志规范
```

制定两条强制执行的文档规范：
1. **UTF-8 声明**：所有 `README.md` 第一行写 `<!-- 编码：UTF-8 -->`
2. **文件命名**：非 `README.md` 的 `.md` 文件优先中文命名，其次小写英文，禁止全大写（`README.md` 除外）
