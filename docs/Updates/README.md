# 📋 StudySolo 更新日志

本目录记录 StudySolo 平台的每日更新内容，面向用户说明新增功能、体验优化与问题修复。

---

## 目录索引

每天创建一个以日期命名的 Markdown 文件，格式为 `YYYY-MM-DD.md`。

| 文件 | 日期 | 版本 / 里程碑 |
|------|------|--------------|
| [2026-02-24.md](./2026-02-24.md) | 2026 年 2 月 24 日 | **🏗️ 项目立项** v0.1.001 — 技术栈确立 · 架构规划 |

---

## 编写规范

1. **每完成一个工作流**后，按时间顺序追加一条更新记录
2. **分类明确**：每次更新需要分为【面向用户】和【面向开发者】两个部分。先写【面向用户】，再写【面向开发者】。
3. **面向用户**：用通俗语言说明"改了什么、用户能感受到什么"，避免开发术语。对于一些不方便和用户说的技术细节或内部修复，一律使用"修复了若干已知问题"作为占位代替。
4. **序号对应一致**：无论是向用户公开的功能还是仅向开发者说明的修复，两部分的序号必须完全一一对应。如果某项更新只对开发者说明，则在【面向用户】中对应序号的内容写为"修复了若干已知问题"。
5. **版本号格式**：采用语义化版本号 `a.b.c` 格式进行编号（例如 `0.1.001`）。
   - **a (大版本)**：超大顶尖更新变更（除非明确声明，否则默认不改变）。
   - **b (小版本)**：极大的改动（如新增一整个新功能），变更门槛较高。
   - **c (补丁版本)**：平日小改动、修复或优化，每次更新递增（三位数，如 001）。
6. **格式统一**：每条记录包含简短标题 + 详细说明。
7. **文件命名**：日期文件直接用 `YYYY-MM-DD.md`；若同一天有多个主题，可追加描述，如 `2026-02-24-架构规划.md`。
