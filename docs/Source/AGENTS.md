# 构建可访问、快速、愉悦的用户界面简明规则

## 设计与实现
*   **必须**：优先使用项目中已配置的 Shadcn UI 组件库。这些组件已为可访问性、设计一致性和交互行为进行了预优化。
*   **必须**：色彩应优先使用 globals.css 中通过 @theme 指令配置的品牌色变量，以确保障碍统一性并便于主题切换。
*   **必须**：当 Shadcn UI 组件库中缺少所需组件时，应基于其设计范式和 CSS 变量体系进行构建，或谨慎评估后将其贡献回组件库。

## 交互

### 键盘
*   **必须**：根据 [WAI-ARIA APG](https://www.w3.org/WAI/ARIA/apg/patterns/) 提供完整的键盘支持
*   **必须**：提供可见的焦点环（`:focus-visible`；使用 `:focus-within` 组合）
*   **必须**：根据APG模式管理焦点（陷阱、移动、返回）
*   **永不**：使用 `outline: none` 而不提供可见的焦点替代方案

### 目标区域与输入
*   **必须**：点击目标≥24像素（移动端≥44像素）；若视觉区域小于24像素，则扩展点击区域
*   **必须**：移动端 `<input>` 的字体大小≥16像素，防止iOS自动缩放
*   **永不**：禁用浏览器缩放（`user-scalable=no`，`maximum-scale=1`）
*   **必须**：设置 `touch-action: manipulation` 以防止双击缩放
*   **应该**：设置 `-webkit-tap-highlight-color` 以匹配设计

### 表单
*   **必须**：表单输入具有水合安全性（焦点/值不会丢失）
*   **永不**：阻止在 `<input>`/`<textarea>` 中粘贴
*   **必须**：加载中的按钮显示旋转指示器并保留原始标签
*   **必须**：在 `<textarea>` 中，Enter键换行；⌘/Ctrl+Enter提交；其他情况下Enter键提交获得焦点的输入项
*   **必须**：保持提交按钮启用，直到请求开始；然后禁用并显示旋转指示器
*   **必须**：接受自由文本输入，后进行验证——不要阻止输入
*   **必须**：允许提交不完整的表单，以触发验证提示
*   **必须**：错误信息内联显示在字段旁；提交时，焦点移至第一个错误处
*   **必须**：使用 `autocomplete` 和有意义的 `name`；正确的 `type` 和 `inputmode`
*   **应该**：为邮箱/验证码/用户名禁用拼写检查
*   **应该**：占位符以 `…` 结尾并展示示例格式
*   **必须**：导航离开前，警告未保存的更改
*   **必须**：兼容密码管理器和双因素验证；允许粘贴验证码
*   **必须**：修剪值以处理文本扩展带来的尾部空格
*   **必须**：复选框/单选按钮无死区；标签+控件共享一个点击目标

### 状态与导航
*   **必须**：URL反映状态（可深层链接的过滤器/标签页/分页/展开面板）
*   **必须**：前进/后退能恢复滚动位置
*   **必须**：导航链接使用 `<a>`/`<Link>`（支持Cmd/Ctrl/中键点击）
*   **永不**：使用 `<div onClick>` 进行导航

### 反馈
*   **应该**：使用乐观UI；根据响应进行协调；失败时回滚或提供“撤销”选项
*   **必须**：确认破坏性操作或提供“撤销”窗口
*   **必须**：使用礼貌的 `aria-live` 区域处理提示信息/内联验证
*   **应该**：对需后续操作的选项使用省略号“…”（如“重命名…”），对加载状态也使用（如“加载中…”）

### 触控与拖拽
*   **必须**：足够大的目标区域，清晰的示能性；避免精细复杂的交互
*   **必须**：首次显示工具提示延迟；后续同类提示即时显示
*   **必须**：在模态框/抽屉中使用 `overscroll-behavior: contain`
*   **必须**：拖拽期间，禁用文本选择并对被拖拽元素设置 `inert`
*   **必须**：如果看起来可点击，就必须是可点击的

### 自动聚焦
*   **应该**：桌面端对单个主要输入项使用自动聚焦；移动端慎用

## 动画
*   **必须**：尊重 `prefers-reduced-motion`（提供简化版本或禁用动画）
*   **应该**：优先CSS > Web Animations API > JS库
*   **必须**：仅动画合成器友好属性（`transform`，`opacity`）
*   **永不**：动画布局属性（`top`，`left`，`width`，`height`）
*   **永不**：使用 `transition: all`——明确列出属性
*   **应该**：仅动画以阐明因果关系或增加精心设计的愉悦感
*   **应该**：根据变化（大小/距离/触发方式）选择缓动效果
*   **必须**：动画可中断且由输入驱动（无自动播放）
*   **必须**：设置正确的 `transform-origin`（运动应从其“物理上”应开始的地方开始）
*   **必须**：SVG变换应用于带 `transform-box: fill-box` 的 `<g>` 包装器

## 布局
*   **应该**：视觉对齐；当感知优于几何时，调整±1像素
*   **必须**：有意识地与网格/基线/边缘对齐——避免随意放置
*   **应该**：平衡图标/文本组合（权重/大小/间距/颜色）
*   **必须**：验证移动端、笔记本电脑、超宽屏（可通过50%缩放模拟超宽屏）
*   **必须**：尊重安全区域（`env(safe-area-inset-*)`）
*   **必须**：避免不必要的滚动条；修复溢出问题
*   **应该**：优先使用弹性/网格布局，而非JS测量布局

## 内容与可访问性
*   **应该**：优先提供内联帮助；工具提示作为最后手段
*   **必须**：骨架屏应反映最终内容以避免布局偏移
*   **必须**：`<title>` 匹配当前上下文
*   **必须**：无死胡同；始终提供下一步/恢复选项
*   **必须**：设计空状态/稀疏状态/密集状态/错误状态
*   **应该**：使用弯引号（“ ”）；避免孤行/寡行（`text-wrap: balance`）
*   **必须**：数字比较时使用 `font-variant-numeric: tabular-nums`
*   **必须**：状态提示应具有冗余性（不单靠颜色）；图标应有文本标签
*   **必须**：即使视觉上省略了标签，也要存在可访问的名称
*   **必须**：使用 `…` 字符（而非 `...`）
*   **必须**：为标题设置 `scroll-margin-top`；提供“跳转到内容”链接；使用层级化 `<h1>`–`<h6>`
*   **必须**：对用户生成的内容具有弹性（短/平均/非常长）
*   **必须**：使用符合区域设置的日期/时间/数字格式（`Intl.DateTimeFormat`，`Intl.NumberFormat`）
*   **必须**：准确的 `aria-label`；装饰性元素使用 `aria-hidden`
*   **必须**：纯图标按钮需有描述性的 `aria-label`
*   **必须**：优先使用原生语义元素（`button`，`a`，`label`，`table`），再考虑ARIA
*   **必须**：使用不换行空格：`10&nbsp;MB`，`⌘&nbsp;K`，品牌名称

## 内容处理
*   **必须**：文本容器能处理长内容（`truncate`，`line-clamp-*`，`break-words`）
*   **必须**：弹性盒子项需要 `min-w-0` 以实现截断
*   **必须**：处理空状态——空字符串/数组不会导致UI损坏

## 性能
*   **应该**：在iOS低功耗模式和macOS Safari上测试
*   **必须**：可靠地测量（禁用可能影响运行时的扩展程序）
*   **必须**：跟踪并最小化重新渲染（使用React DevTools/React Scan）
*   **必须**：在CPU/网络节流条件下进行分析
*   **必须**：批量读写布局；避免回流/重绘
*   **必须**：变更操作（`POST`/`PATCH`/`DELETE`）目标耗时<500毫秒
*   **应该**：优先使用非受控输入；受控输入应确保每次按键输入开销低廉
*   **必须**：虚拟化大型列表（>50项）
*   **必须**：预加载首屏图片；懒加载其余图片
*   **必须**：防止累积布局偏移（明确图片尺寸）
*   **应该**：对CDN域名使用 `<link rel="preconnect">`
*   **应该**：关键字体：`<link rel="preload" as="font">` 配合 `font-display: swap`

## 深色模式与主题
*   **必须**：深色主题在 `<html>` 上设置 `color-scheme: dark`
*   **应该**：`<meta name="theme-color">` 匹配页面背景
*   **必须**：原生 `<select>`：明确设置 `background-color` 和 `color`（修复Windows问题）

## 水合作用
*   **必须**：带 `value` 的输入项需要 `onChange`（或使用 `defaultValue`）
*   **应该**：保护日期/时间渲染，防止水合作用不匹配

## 设计
*   **应该**：使用分层阴影（环境光 + 直射光）
*   **应该**：通过半透明边框+阴影实现清晰边缘
*   **应该**：嵌套圆角：子元素 ≤ 父元素；同心圆
*   **应该**：色调一致性：边框/阴影/文本颜色向背景色调靠拢
*   **必须**：可访问的图表（使用色盲友好配色方案）
*   **必须**：满足对比度要求——优先使用[APCA](https://apcacontrast.com/)而非WCAG 2
*   **必须**：在 `:hover`/`:active`/`:focus` 时增加对比度
*   **应该**：浏览器UI颜色与背景相匹配
*   **应该**：避免深色渐变产生条带（必要时使用背景图片）
