## 核心原则

1. **先冻结再实施**：Phase 0-1 是硬性 Gate，不可跳过
2. **单一事实源**：节点系统只有后端 manifest 作为权威
3. **四层兼容性**：子后端不只是字段兼容
4. **Wiki 是发布源**：不是设计源
5. **每个 Task 独立 PR**：出问题可单独 revert
