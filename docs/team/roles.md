# 团队分工与职责

> 最后更新：2026-04-09
> 项目：StudySolo — 智能学习工作流平台

## 团队概况

StudySolo 采用 **3 人精干团队** 模式运作，以队长主线驱动 + 队员专项负责的方式实现高效并行开发。

### 成员与职责

| 角色 | 成员 | 主要职责 | 负责模块 |
|------|------|----------|----------|
| **队长** | 羽升 | 主线开发、架构决策、Code Review | 主后端、主前端、工作流引擎、AI 路由、部署运维 |
| **小李** | — | 子 Agent 开发 | 子后端 Agent 服务、Agent Gateway、OpenAI 兼容接入 |
| **小陈** | — | Wiki 与用户研究 | Wiki 站点、用户文档、API 文档、社区节点文档；用户调研（问卷设计、投放、数据分析与结论沉淀） |

### 全员共同职责

> **所有成员**均有义务积极发现、上报项目漏洞，尤其是安全问题。

- 发现问题时，必须在 GitHub Issues 中提交规范化的 Issue
- 安全类问题使用 `🔴 security` 标签，优先级最高
- 参见 [Issue 管理规范](issue-management.md)
