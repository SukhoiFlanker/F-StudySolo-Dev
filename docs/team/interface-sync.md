# 接口同步规范

> 最后更新：2026-04-09

团队并行开发时，各模块之间的接口必须保持统一。

## 1. 接口冻结原则

> **先冻结契约，再并行开发。** 多人协作前必须冻结以下接口：

| 接口类型 | 冻结内容 | 责任人 |
|----------|----------|--------|
| API Schema | RESTful 路由、请求/响应体 | 羽升 |
| 节点 Manifest | 节点类型、字段定义、执行协议 | 羽升 |
| Agent Gateway | 子后端注册/发现/调用契约 | 羽升 + 小李 |
| Wiki 文档源 | 文档来源规则、同步边界 | 羽升 + 小陈 |

## 2. 子后端 Agent 接口规范

子后端 Agent 必须遵循 **OpenAI 兼容协议** 的 4 层兼容性：

1. **请求兼容**：遵循 OpenAI Chat Completions API 请求格式
2. **响应兼容**：返回标准 `choices[].message` 结构
3. **运行时兼容**：超时、重试、幂等行为一致
4. **平台治理兼容**：认证、审计、计量统一对接

## 3. Wiki 同步规范

| 原则 | 说明 |
|------|------|
| Wiki 是**发布源** | 面向用户的文档展示，不是设计文档 |
| 主仓是**设计源** | 架构设计、规范文档归属主仓 `docs/` |
| 单向发布 | 主仓设计 → Wiki 发布，不反向 |

Wiki 作为开源项目的**完整对外文档体系**，三大板块正常对外展示：
- **产品文档**：功能介绍、用户指南
- **开发者文档**：API 参考、节点开发指南、插件开发 SOP
- **社区文档**：贡献指南、社区节点发布规范

## 4. 文档先行原则

以下改动在提交代码 PR 前，**必须先有设计文档或 Issue**：

- 新增子后端 Agent
- 新增插件模块
- 新增公共节点类型
- 新增数据库表
- 新增跨项目接口
- 修改已冻结契约
