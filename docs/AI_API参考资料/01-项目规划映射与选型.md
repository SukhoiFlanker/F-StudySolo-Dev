# 项目规划映射与选型

## 一、从项目 docs 提取到的 AI API 事实

依据：
- `docs/Plans/global/PROJECT_PLAN.md`
- `docs/Plans/global/项目深度功能规划.md`
- `docs/Plans/StudySolo-MVP.md`

明确规划：
- 使用 `openai` Python SDK 作为统一调用层。
- 两路模型供应商均走 OpenAI 兼容格式：
  - 火山方舟（doubao 路线）
  - 阿里云百炼（qwen 路线）
- 后端以 SSE 推送逐步执行结果，前端使用 `EventSource` 接收。

## 二、建议的接口优先级（规划阶段）

1. 第一优先：OpenAI Chat Completions 兼容接口
- 原因：两家供应商兼容成熟、迁移成本最低。

2. 第二优先：OpenAI Responses 接口（按需）
- 原因：适合后续 agent 化能力（工具调用、上下文延续）。
- 说明：先保证 MVP 跑通，再逐步迁移到 Responses。

3. Embedding 接口（为语义检索预留）
- 原因：项目规划已出现向量检索需求（pgvector / embedding 字段）。

## 三、统一环境变量建议

```env
# OpenAI 直连（可选）
OPENAI_API_KEY=

# 火山方舟
VOLCENGINE_API_KEY=
VOLCENGINE_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
VOLCENGINE_MODEL=doubao-2.0-pro

# 阿里云百炼（中国内地）
DASHSCOPE_API_KEY=
DASHSCOPE_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
DASHSCOPE_MODEL=qwen3-turbo
```
