# 宝塔面板部署详细指南

> 📅 编写日期：2026-02-25  
> 🎯 适用项目：StudySolo — 基于自然语言的 AI 学习赋能工作流平台  
> 🌐 域名：studyflow.1037solo.com  
> 🏗️ 环境：阿里云 ECS（2核4G）+ 宝塔面板 + Nginx

---

## 部署架构总览

```
用户浏览器
    │
    ▼ https://studyflow.1037solo.com
    │
阿里云 ECS (2核4G · Alibaba Cloud Linux 3)
    │
    ├── 宝塔面板（服务器管理 + SSL + 防火墙）
    │
    ├── Nginx（宝塔托管）
    │     │
    │     ├── 80/443 端口对外
    │     ├── /          → 127.0.0.1:2037  (Next.js 前端)
    │     ├── /api/      → 127.0.0.1:2038  (FastAPI 后端)
    │     └── /_next/    → 127.0.0.1:2037  (静态资源缓存)
    │
    ├── PM2 守护 → Next.js 16.1  (port 2037，仅本地监听)
    │
    └── Gunicorn 守护 → FastAPI    (port 2038，仅本地监听)
```

**核心思路**：利用宝塔"PHP项目（纯静态）"作为一个 **Nginx 网关**，统一接管 80/443 端口。前端和后端各自在内部端口运行，通过 Nginx 反向代理对外暴露。这样做的好处是：

1. **同域同端口** — 前端 `/` 和后端 `/api/` 在同一域名下，浏览器无跨域问题
2. **统一 SSL** — 只需在网关站点申请一次证书即可
3. **隔离运行** — 后端崩溃不影响前端，反之亦然

## 安全性评估

> ✅ **结论：本架构安全性完全满足生产需求。**

| 安全维度 | 评估 | 说明 |
|---------|------|------|
| **端口暴露** | ✅ 安全 | 2037/2038 仅监听 `127.0.0.1`，从外网无法直连，必须经过 Nginx |
| **SSL/HTTPS** | ✅ 安全 | 宝塔一键申请 Let's Encrypt 证书 + 强制 HTTPS 跳转，传输层全加密 |
| **跨域攻击** | ✅ 安全 | 前后端同域（`studyflow.1037solo.com`），无 CORS 问题，也消除了跨域引发的 CSRF 风险 |
| **API Key 泄露** | ✅ 安全 | 所有 AI API Key 仅存在后端 `.env` 中，前端通过 `/api/ai/*` 代理调用，零泄露 |
| **SSE 流式安全** | ✅ 安全 | `proxy_buffering off` 仅关闭了 Nginx 缓冲，不影响安全（数据仍经过 HTTPS 加密） |
| **安全响应头** | ✅ 已配置 | X-Frame-Options / X-Content-Type-Options / X-XSS-Protection / Referrer-Policy |
| **进程隔离** | ✅ 安全 | PM2 和 Gunicorn 分别守护前后端进程，崩溃自动重启，互不干扰 |
| **应用层防护** | ✅ 已规划 | FastAPI 中间件：JWT 鉴权 + Pydantic 输入验证 + slowapi 限流 + CORS 白名单 |
| **宝塔防火墙** | ✅ 可用 | Nginx WAF 插件（免费版）可防 CC 攻击、SQL 注入、XSS |

### ⚠️ 安全必做清单

| 序号 | 操作 | 说明 |
|------|------|------|
| 1 | **阿里云安全组** | 仅开放 80 / 443 / 22 / 宝塔面板端口，其他全部拒绝 |
| 2 | **宝塔面板端口** | 安装后第一件事：改为非默认端口（如 28888）并设 IP 白名单 |
| 3 | **SSH 安全** | 禁用密码登录，仅允许密钥登录；限制来源 IP |
| 4 | **后端 .env** | 绝对不要提交到 Git，`.gitignore` 已配置排除 |
| 5 | **Supabase RLS** | 所有表开启 Row Level Security，确保用户只能访问自己的数据 |

## 详细部署步骤

### 第一步：上传代码到服务器

**方式一：Git 克隆（推荐）**

```bash
# SSH 连接服务器（或通过宝塔终端）
ssh root@<ECS公网IP>

# 创建项目目录
mkdir -p /www/wwwroot/studysolo
cd /www/wwwroot/studysolo

# 克隆代码
git clone <你的仓库地址> .
```

**方式二：宝塔文件管理器**

直接在宝塔面板的"文件"功能中上传压缩包到 `/www/wwwroot/studysolo`，然后解压。

---

### 第二步：创建 Nginx 网关站点

> 对应宝塔顶部的 **"PHP项目"** 选项卡

1. 点击左上角 **"添加站点"**
2. 填写配置：

| 配置项 | 填写内容 |
|--------|---------|
| **域名** | `studyflow.1037solo.com` |
| **根目录** | `/www/wwwroot/studyflow.1037solo.com`（默认即可） |
| **PHP版本** | 选择 **"纯静态"** |
| **数据库** | 不创建 |

3. 站点创建完成后，进入 **设置 → SSL**：
   - 选择 **Let's Encrypt**
   - 勾选域名 `studyflow.1037solo.com`
   - 点击"申请"
   - 申请成功后，开启 **"强制HTTPS"**

> 💡 **为什么是"纯静态"？** 因为这个站点本身不跑任何代码，它只是一个 Nginx 配置容器，充当流量分发网关。

---

### 第三步：配置 Nginx 反向代理（核心）

进入站点 **设置 → 配置文件** ，在 `server {}` 块内的适当位置（删除宝塔默认的静态文件规则后），**写入以下反向代理规则**：

```nginx
    # ==========================================
    # 一、前端 Next.js（将根路径 / 交给 2037 端口）
    # ==========================================
    location / {
        proxy_pass http://127.0.0.1:2037;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # WebSocket 支持（Next.js 开发热更新 + Supabase 实时功能）
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    # ==========================================
    # 二、后端 FastAPI（将 /api/ 交给 2038 端口）
    # ==========================================
    location /api/ {
        proxy_pass http://127.0.0.1:2038/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 🚨【极端重要】以下三行必须配置！
        # 否则 AI 流式输出（SSE）会被 Nginx 缓冲，
        # 用户会看到等待 30 秒后一次性蹦出所有文字，而不是逐字打出
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 300s;  # AI 接口最长等 5 分钟
    }

    # ==========================================
    # 三、Next.js 静态资源终极缓存
    # ==========================================
    location /_next/static/ {
        proxy_pass http://127.0.0.1:2037;
        expires 365d;
        add_header Cache-Control "public, immutable";
    }

    # ==========================================
    # 四、安全响应头
    # ==========================================
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;

    # ==========================================
    # 五、Gzip 压缩 + 请求体限制
    # ==========================================
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml;
    gzip_min_length 1024;
    client_max_body_size 10m;
```

配置完毕后，点击"保存"。宝塔会自动检测 Nginx 语法是否正确。

---

### 第四步：部署前端 Next.js（端口 2037）

> 对应宝塔顶部的 **"Node项目"** 选项卡

#### 4.1 构建前端

```bash
cd /www/wwwroot/studysolo/frontend

# 安装 pnpm（如果宝塔 Node 环境没有）
npm install -g pnpm

# 安装依赖
pnpm install

# 创建环境变量文件
cp .env.example .env.local
# 编辑 .env.local 填入实际的 Supabase Key 等

# 构建生产版本
pnpm build
```

#### 4.2 在宝塔创建 Node 项目

1. 进入 **"Node项目"** 选项卡
2. 点击 **"添加 Node 项目"**
3. 填写配置：

| 配置项 | 填写内容 |
|--------|---------|
| **项目目录** | `/www/wwwroot/studysolo/frontend` |
| **启动命令** | `pnpm start` |
| **项目端口** | `2037` |
| **Node 版本** | `20.18.0 LTS` |
| **绑定域名** | ⚠️ **留空！不绑定任何域名！** |
| **PM2 管理** | ✅ 开启 |
| **开机启动** | ✅ 开启 |

> ⚠️ **关键：不要绑定域名！** 外网访问由第二步的 Nginx 网关负责转发，这里只需在本地 `127.0.0.1:2037` 跑着就行。

---

### 第五步：部署后端 FastAPI（端口 2038）

> 对应宝塔顶部的 **"Python项目"** 选项卡

#### 5.1 准备后端环境

```bash
cd /www/wwwroot/studysolo/backend

# 创建 Python 虚拟环境
python3 -m venv venv-studysolo
source venv-studysolo/bin/activate

# 安装依赖
pip install -r requirements.txt

# 创建环境变量文件
cp .env.example .env
# 编辑 .env 填入所有密钥（Supabase · AI API · JWT 等）
```

#### 5.2 在宝塔创建 Python 项目

1. 进入 **"Python项目"** 选项卡
2. 点击 **"添加 Python 项目"**
3. 填写配置：

| 配置项 | 填写内容 |
|--------|---------|
| **项目目录** | `/www/wwwroot/studysolo/backend` |
| **启动方式** | `ASGI（FastAPI/Starlette）` |
| **启动命令** | `gunicorn app.main:app -c gunicorn.conf.py` |
| **项目端口** | `2038` |
| **Python 版本** | `3.11+` |
| **虚拟环境** | 选择或创建 `venv-studysolo` |
| **绑定域名** | ⚠️ **留空！不绑定任何域名！** |
| **开机启动** | ✅ 开启 |

#### 5.3 Gunicorn 生产配置文件

确认 `backend/gunicorn.conf.py` 内容如下：

```python
# backend/gunicorn.conf.py
bind = "127.0.0.1:2038"
workers = 2                               # 2核 ECS 使用 2 workers
worker_class = "uvicorn.workers.UvicornWorker"
timeout = 300                             # AI 接口最长 5 分钟
graceful_timeout = 120
accesslog = "/www/wwwlogs/studysolo-backend-access.log"
errorlog  = "/www/wwwlogs/studysolo-backend-error.log"
preload_app = True
max_requests = 1000
max_requests_jitter = 50
```

> 💡 `bind = "127.0.0.1:2038"` 确保后端只在本机可访问，外网无法直连。

## 日常更新部署流程

每次代码更新后的部署步骤：

```bash
# 1. 在本地：提交并推送代码
git push origin main

# 2. SSH 连接服务器（或宝塔终端）
ssh root@<ECS_IP>
cd /www/wwwroot/studysolo
git pull origin main

# 3. 更新前端（需要重新编译构建）
cd frontend
pnpm install       # 仅依赖有变化时需要
pnpm build
pm2 restart StudySolo-Frontend

# 4. 更新后端（Python 无需编译，直接重启）
cd ../backend
source venv-studysolo/bin/activate
pip install -r requirements.txt   # 仅依赖有变化时需要
# 方式一：宝塔面板 → Python 项目管理器 → StudySolo-Backend → 重启
# 方式二：命令行
kill -HUP $(cat /tmp/gunicorn-studysolo.pid)  # 优雅重启
```

> 💡 **Python 优势**：后端是普通 Python 脚本，**无需编译构建**。改了一个 `.py` 文件，重启守护进程即可生效，比 Node.js 项目快得多。

## 常见问题排查

### Q1: 访问域名显示宝塔默认页

**原因**：Nginx 配置未生效或 Next.js 未启动。

**排查**：
```bash
# 检查 Next.js 是否在运行
pm2 list
# 检查端口是否被监听
ss -tlnp | grep 2037
```

### Q2: `/api/` 接口返回 502 Bad Gateway

**原因**：FastAPI 后端未启动或 Gunicorn 崩溃。

**排查**：
```bash
# 检查后端日志
tail -50 /www/wwwlogs/studysolo-backend-error.log
# 检查端口
ss -tlnp | grep 2038
```

### Q3: AI 流式输出卡顿（整段一起出现而非逐字）

**原因**：Nginx 的 `proxy_buffering` 没有关闭。

**解决**：确认 `/api/` 的 location 配置中有 `proxy_buffering off;` 和 `proxy_cache off;`。

### Q4: SSL 证书过期

宝塔的 Let's Encrypt 证书默认 90 天，宝塔会自动续期。如果续期失败：
- 检查域名 DNS 是否正确指向 ECS IP
- 手动到站点设置 → SSL → 重新申请

## 服务器资源监控

### 2核4G 内存分配预估

```
系统 + 宝塔面板      ~800 MB
Nginx                ~50 MB
Next.js (PM2)        ~800 MB - 1.2 GB
FastAPI (Gunicorn)   ~300 MB - 500 MB
系统缓存/Buffer      ~1 GB
预留                  ~500 MB
─────────────────────────────
总计                  ~3.5 GB (有余量)
```

### Swap 设置（建议）

```bash
# 设置 2GB Swap 作为内存兜底
fallocate -l 2G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile
echo '/swapfile none swap sw 0 0' >> /etc/fstab
```
