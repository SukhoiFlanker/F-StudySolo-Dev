## 2. 节点插件

### 2.1 节点分类

| 类型 | 说明 | 复杂度 | 开发文件数 |
|------|------|--------|----------|
| A 型 | 纯 LLM Prompt 节点 | 低 | 2-3 个文件 |
| B 型 | 外部工具节点 | 中 | 3-5 个文件 |
| C 型 | 全功能插件（社区节点） | 高 | 10+ 个文件 |

### 2.2 节点目录结构

**标准节点**：
```
backend/app/nodes/
├── _base.py                    # BaseNode 抽象基类（运行时权威）
├── _categories.py              # 节点分类枚举
├── _mixins.py                  # 可复用 Mixin
├── __init__.py                 # 节点注册表
├── [category]/                 # 分类目录
│   ├── [node_type]/           # 节点目录
│   │   │   ├── node.py        # 节点实现
│   │   │   └── __init__.py
│   └── ...
└── community/                  # 社区节点
    └── node.py
```

**节点分类**：

| 分类 | 说明 | 示例 |
|------|------|------|
| `input` | 输入节点 | `trigger_input`, `knowledge_base`, `web_search` |
| `analysis` | 分析/控制节点 | `ai_analyzer`, `ai_planner`, `logic_switch`, `loop_map` |
| `generation` | 内容生成节点 | `outline_gen`, `content_extract`, `summary`, `flashcard`, `quiz_gen`, `mind_map`, `compare`, `merge_polish` |
| `interaction` | 交互节点 | `chat_response` |
| `output` | 输出节点 | `export_file`, `write_db` |
| `structure` | 结构节点 | `loop_group`（前端虚拟容器） |
| `community` | 社区节点 | 用户发布的节点 |

### 2.3 节点开发流程

**Step 0：分类判断**

参考 `docs/项目规范与框架流程/功能流程/新增AI工具/00-节点与插件分类判断.md`

**Step 1：创建节点目录**

```bash
# A/B 型节点
backend/app/nodes/<category>/<node_type>/
├── node.py
└── __init__.py

# C 型或社区节点
backend/app/nodes/community/<node_type>/
├── node.py
├── plugin.json
├── README.md
└── __init__.py
```

**Step 2：实现节点类**

```python
# backend/app/nodes/generation/quiz_gen/node.py
from app.nodes._base import BaseNode, NodeInput

class QuizGenNode(BaseNode):
    node_type = "quiz_gen"
    category = "generation"
    description = "生成测验题"
    is_llm_node = True
    output_format = "json"

    async def execute(self, node_input: NodeInput, llm_caller):
        messages = self.build_messages(node_input)
        async for token in llm_caller(messages):
            yield token

    async def post_process(self, raw_output: str):
        data = json.loads(raw_output)
        return NodeOutput(content=json.dumps(data, ensure_ascii=False), format="json")
```

**Step 3：注册到 Manifest**

节点通过 `backend/app/nodes/__init__.py` 的注册表自动暴露给前端。

### 2.4 plugin.json 规范（C 型/社区节点）

```json
{
  "id": "anki-exporter",
  "name": "Anki Exporter",
  "version": "1.0.0",
  "author": "Team Member",
  "node_type": "anki_export",
  "category": "output",
  "description": "Export flashcards to Anki format",
  "dependencies": [],
  "frontend": {
    "renderer": "AnkiExportRenderer"
  }
}
```

## 3. 前端插件

### 3.1 插件目录结构

```
frontend/src/plugins/
├── registry.ts              # 插件注册表
├── README.md
└── my-custom-plugin/        # 插件示例
    ├── plugin.json
    ├── renderer.tsx
    └── node-config.ts
```

### 3.2 插件注册

```typescript
// frontend/src/plugins/registry.ts
interface Plugin {
  id: string;
  name: string;
  nodeTypes: string[];
  renderer: React.ComponentType;
}

const plugins: Plugin[] = [];

export function registerPlugin(plugin: Plugin) {
  plugins.push(plugin);
  RENDERER_REGISTRY[plugin.nodeTypes[0]] = plugin.renderer;
}

export function getPlugins(): Plugin[] {
  return [...plugins];
}
```
