## 1. 产品定位

**StudySolo** 是 AI 学习工作流引擎——用户用自然语言描述学习目标，系统自动生成可视化节点 DAG，按拓扑顺序执行并流式推送结果。

```
用户输入 "帮我学机器学习"
     ↓
[AI 规划] → 生成节点骨架: trigger_input → ai_analyzer → outline_gen → summary → flashcard
     ↓
[执行引擎] → DAG 拓扑排序 → 逐节点 SSE 流式执行
     ↓
[前端画布] → 实时展示节点状态变化 + token 输出
```

**不是**：
- 通用聊天机器人
- LangChain 封装套壳
- 传统在线教育平台

**是**：
- 可视化 DAG 编辑器 + AI 执行引擎
- 意图驱动的学习流程编排器
- 插件化节点生态

## 4. 节点体系

### 4.1 节点分类

| 分类 | 节点类型 | 说明 |
|------|----------|------|
| **输入 (Input)** | `trigger_input` | 工作流入口，接收用户目标 |
| | `knowledge_base` | 从知识库检索相关内容 |
| | `web_search` | 联网搜索外部资料 |
| **分析 (Analysis)** | `ai_analyzer` | 深度分析输入内容 |
| | `ai_planner` | 生成后续执行步骤 |
| | `logic_switch` | 条件分支 |
| | `loop_map` | 列表循环处理 |
| **生成 (Generation)** | `outline_gen` | 生成结构化大纲 |
| | `content_extract` | 提取关键信息 |
| | `summary` | 生成摘要 |
| | `flashcard` | 生成记忆卡片 |
| | `quiz_gen` | 生成测验题 |
| | `compare` | 对比分析 |
| | `mind_map` | 思维导图 |
| | `merge_polish` | 合并润色 |
| **交互 (Interaction)** | `chat_response` | 用户对话回复 |
| **输出 (Output)** | `write_db` | 写入数据库 |
| | `export_file` | 导出文件 |
| **结构 (Structure)** | `loop_group` | 循环容器 |

### 4.2 Manifest-First 原则

- **后端 manifest 是节点定义的唯一事实源**
- 前端 `RENDERER_REGISTRY` 从后端 manifest 动态读取
- `workflow-meta.ts` 继续承担 `status / icon / theme / inputs / outputs` 等结构职责
- **不要删除** `workflow-meta.ts`（Phase 3 确认的冻结边界）

## 9. 关键工程约束

### 9.1 前端目录约束

```
src/
├── app/                # 路由入口层
├── features/           # 业务域模块（workflow/admin/auth/knowledge/settings）
├── components/         # 仅通用 UI 与 layout
├── hooks/              # 仅全局通用 hook
├── stores/             # Zustand 状态层
├── services/           # 请求层
├── types/              # 领域类型
├── lib/                # 第三方库配置（events/event-bus.ts）
└── styles/             # 全局样式
```

### 9.2 行数治理

- 长期目标：`src/**/*.ts|tsx|css <= 300` 行
- 当前执行策略：**ratchet**
  - 禁止新增超限文件
  - 禁止历史超限文件继续变大
- 页面默认目标：`app/**/page.tsx <= 220` 行

### 9.3 事件通信

- 新代码必须使用 **TypedEventBus**（`frontend/src/lib/events/event-bus.ts`）
- 旧 CustomEvent 在涉及该文件的 PR 中逐步替换

### 9.4 状态与副作用

- `stores/*` 只负责状态和纯更新函数
- 禁止在 store 内直接执行网络请求、路由跳转、定时器或 DOM 操作
- **跨 store 同步必须在调用方显式处理**
