## 答辩前必读 · 备查区（不念，追问时翻阅）

### 仓库与版本

- **Polyglot Monorepo**：`frontend/`（Next.js 16，端口 2037）、`backend/`（FastAPI，2038）、`agents/`（多微服务 8001–8099）、`introduce/`（Vite 介绍页）、`shared/`（Git Submodule）、`supabase/migrations/`。
- **竞赛提交版**冻结于 GitHub `AIMFllys/StudySolo`；**本仓库**为活跃开发分支。

### 前端核心路径

- 画布：`@xyflow/react` + Zustand（`use-workflow-store`），Undo/Redo、执行追踪 UI。
- 画布 → AI：`use-canvas-context.ts` 的 `serialize()` 生成 `CanvasContext`（节点摘要、DAG 文本、选中节点、坐标），由 `SidebarAIPanel` → `useStreamChat` 发送。**不含**全站路由自动感知。
- 三种模式：Chat / Plan / Create；Create 下区分 BUILD（空画布 → `/api/ai/generate-workflow`）与 MODIFY（增量 JSON + 重试）。
- 统一流式入口：`/api/ai/chat-stream`，实现于 `backend/app/api/ai/chat.py`。

### 后端核心路径

- DAG 引擎：`backend/app/engine/executor.py` → `topology` / `level_runner` / `node_runner` / `loop_runner`。
- SSE 事件：`node_input` → `node_status` → `node_token` → `node_done` → `loop_iteration` → `workflow_done`。
- 执行 API：`POST /workflow/{id}/execute`（`backend/app/api/workflow/execute.py`）。
- 节点注册：`backend/app/nodes/__init__.py` 自动发现 → `NODE_REGISTRY`。
- Prompt 体系：`backend/app/prompts/` 模块化 Markdown + 模板变量。

### 节点口径

- 参赛说明书写 **18 种**（冻结版）；当前 `frontend/src/types/workflow.ts` 枚举 **20 项**（含 `community_node`、`loop_map`）。口播说 **「20 种节点类型」** 或 **「18 种学习向 + 控制流与社区扩展」** 均可。

### 社区 · 协作 · 子 Agent

- Fork：`backend/app/api/workflow/social.py` — `POST /{id}/fork`。
- 协作：`backend/app/api/workflow/collaboration.py` — 邮箱邀请、editor/viewer、共享列表。
- 子 Agent：`agents/` 微服务 + `POST /agents/{name}/chat`。

### 数据与安全

- Supabase PostgreSQL + Auth + **RLS**；后台审计。

### 路线图口径（与说明书 7.3 一致）

- MCP：**开发中**；CLI：**规划**；全站 Agent：**规划**。

## P1 · 开场：一个场景，点燃共鸣（约 45s）

**【PPT 提示】**：一张「期末两周倒计时」的场景插画或截图；旁列六个痛点关键词，不要编号。

**口播：**

各位评委老师好，我们是 StudySolo。

先讲一个几乎每位大学生都经历过的场景：**期末还剩两周**，你翻出课件，打开 ChatGPT 问重点，又切到笔记软件整理，再找题库自测——搜索、整理、提问、测验分散在四五个工具里；AI 一次只回答一个问题，前一步的输出到后一步全靠你自己复制粘贴。**过程看不见、结果留不住、流程不可复用**。我们在用户调研中发现，这些困扰可以收束为六类：路径不清、工具割裂、过程黑盒、结果难复用、提问门槛高、效果难评估——从期末复习到考研备考，从技能突击到论文季，**反复出现**。

StudySolo 就是为了把这条碎片链路**拼成一条可编排、可执行、全程可见的自动化流程**。

## P2 · 我们做什么 + 凭什么是我们（约 55s）

**【PPT 提示】**：上半页一句话定位；下半页三列竞品对比表（ChatGPT、Dify/Coze、StudySolo）。

**口播：**

一句话定位：**用自然语言说出学习目标，AI 在画布上生成多节点工作流，按依赖关系逐步执行，全程流式可见**。工作流可以保存、再运行、发到社区，他人一键 Fork 再改——**学习过程本身变成可复用的资产**。

跟现有方案对比，差异集中在三个维度：和 ChatGPT 等通用聊天比，我们把**单轮问答拉成了多步可编排流程**；和 Dify、Coze 等通用工作流比，我们提供**学习场景专属节点**——大纲、闪卡、测验、思维导图等，用户不需要从零搭 DAG；和手动拼多个工具比，**一张画布完成检索、分析、生成、自测和导出，流程一次配好反复跑**。

## P3 · 架构：三层讲清楚（约 70s）

**【PPT 提示】**：一张分层架构图（意图层 → 编排层 → 执行与路由层），右侧小字列技术栈。

**口播：**

整个系统分三层。

**第一层，意图理解与画布交互**。前端基于 Next.js 和 React Flow 搭建可视化画布；左侧 AI 面板在每次对话时，把**画布当前状态**——节点、连线、坐标、上下游关系——序列化为结构化文本发给后端。后端统一走流式接口，先做意图分类：你是闲聊、要改进建议、还是要直接改图？如果画布是空的还能整张图从零生成。**AI 看见的是画布，不是全站**——全站级辅助在路线图里。

**第二层，DAG 编排与执行引擎**。我们自研了拓扑排序 + 按层调度的执行器，支持循环、条件分支。每个节点独立执行，通过 SSE 把状态、流式 Token、最终结果实时推到前端——**用户能看到每一步在做什么，而不是等一个黑盒答案**。

**第三层，多模型路由与数据治理**。后端对接八个 AI 平台、多种路由策略，单路故障自动降级。数据层用 Supabase PostgreSQL，行级安全策略做多租户隔离，配合 JWT 认证与后台审计。

## P4 · 演示：最短闭环（约 55s）

**【PPT 提示】**：录屏截帧或现场操作：工作区 → 输入 → 画布生成 → 运行 → 执行追踪。

**口播：**

演示走一条最短闭环。打开工作台，左侧对 AI 说「帮我做一门课的期末复习流」——画布上出现分析、规划、大纲、总结、闪卡、测验等节点，自动连好线。如果想调整，直接说「把闪卡移到测验后面」，AI 就用结构化指令改画布，支持撤销。点击运行，右侧执行面板能看到每个节点从待执行到完成，流式输出实时滚动。跑完的工作流可以保存、下次直接复用，也可以发布到社区——别人 Fork 一份就能在自己空间改。

---

## P5 · 创新点与壁垒（约 45s）

**【PPT 提示】**：四个图标 + 底部一句「飞轮」。

**口播：**

创新点四条。**节点即智能体**：每个节点有独立 Prompt、独立模型路由和输入输出契约，多节点组成的 DAG 就是一套多智能体协作系统。**多平台模型路由与容灾**：按任务选型，单路故障自动切换。**Prompt 工程化**：提示词以模块化 Markdown 管理，可版本迭代、Git 追踪，而不是散在代码里的字符串。**社区共建加 SOP 驱动扩展**：工作流和节点可共享；团队把「新增节点」写成了标准 SOP 与 AI 技能文档，新增一类节点后端不超过三个文件、前端不超过四个文件。

这四条加在一起形成一个**飞轮**：社区模板和节点越多，新用户上手成本越低，贡献意愿越高，平台能力就越滚越大。

---

## P6 · 路线图：已有什么、下一步做什么（约 45s）

**【PPT 提示】**：左栏「已上线」，右栏「推进中/规划」。

**口播：**

目前**已上线并稳定运行**的：DAG 引擎、20 种节点类型、自然语言生成与修改工作流、社区共享与 Fork、多模型路由、管理后台与计费体系。平台已部署在线，域名 StudyFlow.1037solo.com。

**推进中**的三条线：第一，**MCP 协议接入**，让外部工具与数据以标准协议挂入执行链路；第二，**左侧 Agent 从画布扩展到全站**——在权限可控前提下，帮用户完成 Fork、整理、重命名等跨页面操作；第三，**团队协作深化**，在已有协作者 API 上，做共享编辑、任务分工与工作量可视化，服务小组学习场景。更远期还有面向极客的 CLI 与自动化接入。

## P7 · 团队与收尾（约 40s）

**【PPT 提示】**：三人头像 + 职责关键词 + 二维码/URL。

**口播：**

团队三人。**羽升**，队长，负责核心架构、工作流引擎、AI 路由与线上部署运维，也是全栈主力。**小李**，负责子 Agent 服务与 Gateway，把独立的 AI 能力模块化接入主平台。**小陈**，负责用户侧文档与 Wiki，同时主导用户调研——从问卷设计到数据分析，把真实痛点持续反馈到开发优先级。

最后一句带走：**StudySolo 不是又一个 AI 聊天框，而是让学习智能体能被创建、执行、分享和持续进化的开放平台**。

线上体验 StudyFlow.1037solo.com，开源在 GitHub。感谢各位评委老师！
