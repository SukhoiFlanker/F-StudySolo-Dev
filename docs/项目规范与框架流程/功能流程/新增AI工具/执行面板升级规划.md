## 一、目标与验收标准

### 1.1 功能目标

| 目标 | 验收标准 |
|------|---------|
| 全局执行进度可见 | 执行时右侧自动弹出面板，顶部进度条实时更新（N/M 步骤完成 + 总耗时） |
| 推理链样式展示 | 每个步骤以连续时间线呈现，有左侧竖轴、步骤头固定、流式 token 实时追加 |
| 执行中自动聚焦 | 当前 running 步骤自动展开 + 面板内自动滚动到该步骤 |
| 节点 Input 可读化 | 输入区显示"来自 {节点名称}"而非原始 ID；显示人类可读摘要而非原始 JSON |
| 节点 Output 双视图 | 画布节点 slip 和执行面板 trace item 均可查看 Output，内容一致 |
| 并行执行可视化 | 同一并行组的节点在时间轴上同纵向对齐，有分叉线标识 |
| SOP 闭环 | 新增节点 SOP 包含"面板渲染"验收步骤和 compact 渲染器规范 |

### 1.2 不在本次范围内

- 后端引擎改造（执行逻辑不变）
- 执行历史持久化（本次只做运行时内存 Trace）
- 移动端执行面板（桌面端优先）

## 二、数据层设计（前端类型系统）

### 2.1 新增核心类型

在 `frontend/src/types/workflow.ts` 末尾追加：

```typescript
// ── 执行追踪（Execution Trace）──────────────────────────────────

/**
 * 单个节点的执行追踪记录。
 * 由 use-workflow-execution.ts 在 SSE 事件驱动下维护。
 * 由 ExecutionTraceDrawer 消费展示。
 */
export interface NodeExecutionTrace {
  nodeId: string;
  nodeType: string;
  nodeName: string;           // 来自 WorkflowNode.data.label
  category: string;           // 来自 workflow-meta.ts getNodeTheme()
  status: NodeStatus;

  // 时序
  executionOrder: number;     // 拓扑顺序（1-based）
  startedAt?: number;         // performance.now()
  finishedAt?: number;
  durationMs?: number;

  // 并行执行
  isParallel: boolean;
  parallelGroupId?: string;   // 同 parallelGroupId 的节点同批执行

  // 内容
  inputSummary?: string;      // 从 input_snapshot 提取的人类可读摘要
  rawInputSnapshot?: string;  // 原始 JSON，供"查看原始输入"展开
  streamingOutput: string;    // 流式 token 累积（执行中持续追加）
  finalOutput?: string;       // node_done 后的最终输出
  outputFormat?: string;      // 来自 nodeData.output_format
  errorMessage?: string;

  // 系统信息
  modelRoute?: string;        // 使用的模型路由（来自 nodeData.model_route）
}

/**
 * 一次完整的工作流执行会话。
 * 每次点击"运行全部"创建一个新 session。
 */
export interface WorkflowExecutionSession {
  sessionId: string;
  workflowId: string;
  startedAt: number;          // performance.now()
  finishedAt?: number;
  totalDurationMs?: number;
  overallStatus: 'running' | 'completed' | 'error';
  traces: NodeExecutionTrace[]; // 不保证有序，按 executionOrder 排序显示
  completedCount: number;     // done + skipped + error 节点数
  totalCount: number;         // 所有节点数（不含 trigger_input）
}
```

### 2.2 SSE 事件类型扩展

`frontend/src/types/workflow-events.ts` 无需改动（现有事件已足够）。

但需要新增一个**工具函数**，将 `input_snapshot` 转化为可读摘要：

```typescript
// frontend/src/features/workflow/utils/trace-helpers.ts （新建文件）

import type { NodeExecutionTrace } from '@/types';

/**
 * 将 input_snapshot（原始 JSON 字符串）解析为人类可读摘要。
 * 规则：
 *   - user_content → 取前 60 字符，加 "任务: " 前缀
 *   - upstream_outputs → "接收了 N 个上游输出"
 *   - 两者拼接为 " · " 分隔的摘要行
 */
export function parseInputSummary(rawSnapshot: string | undefined): string {
  if (!rawSnapshot) return '';
  try {
    const parsed = JSON.parse(rawSnapshot);
    const parts: string[] = [];
    if (parsed.user_content) {
      const trimmed = String(parsed.user_content).trim().slice(0, 60);
      parts.push(`任务: ${trimmed}${trimmed.length === 60 ? '…' : ''}`);
    }
    const upCount = Object.keys(parsed.upstream_outputs ?? {}).length;
    if (upCount > 0) {
      parts.push(`接收了 ${upCount} 个上游输出`);
    }
    return parts.join(' · ');
  } catch {
    return '';
  }
}

/**
 * 从 nodeId 列表中提取并行组 ID。
 * 同一批 parallel 执行的节点具有相同的 parallelGroupId（用 sorted nodeIds 生成）。
 */
export function buildParallelGroupId(nodeIds: string[]): string {
  return [...nodeIds].sort().join('|');
}

/**
 * 格式化持续时间（毫秒 → 可读字符串）
 */
export function formatDuration(ms: number): string {
  if (ms < 1000) return `${ms}ms`;
  return `${(ms / 1000).toFixed(1)}s`;
}
```

## 三、Store 层改造

### 3.1 `use-workflow-store.ts` 改造

在现有 Store 中新增 `executionSession` 状态及 5 个操作方法。

**改动范围**：仅新增字段和方法，原有所有字段和方法保持不变（向后兼容）。

```typescript
// 新增 imports
import type { WorkflowExecutionSession, NodeExecutionTrace } from '@/types';
import { parseInputSummary } from '@/features/workflow/utils/trace-helpers';

// 在 WorkflowStore interface 末尾新增：

interface WorkflowStore {
  // ... 已有 ...

  /** 当前执行会话（null 表示未执行或已清除） */
  executionSession: WorkflowExecutionSession | null;

  /** 初始化新执行会话，必须在 SSE 连接前调用 */
  startExecutionSession: (workflowId: string) => void;

  /** 添加一个新的 trace 条目（node_input 事件触发）*/
  registerNodeTrace: (nodeId: string, order: number, isParallel: boolean, parallelGroupId?: string) => void;

  /** 更新某节点的 trace（通用，支持部分更新）*/
  updateNodeTrace: (nodeId: string, updates: Partial<NodeExecutionTrace>) => void;

  /** 流式追加 token（node_token 事件触发，高频调用，更新 streamingOutput）*/
  appendNodeTraceToken: (nodeId: string, token: string) => void;

  /** 结束执行会话 */
  finalizeExecutionSession: (status: 'completed' | 'error') => void;

  /** 清除执行会话（手动关闭或重新执行前调用）*/
  clearExecutionSession: () => void;
}

// 实现（在 create() 的 set/get 中）：

startExecutionSession: (workflowId) => {
  const nodes = get().nodes;
  // 为所有节点预建空 trace（只有 status='pending' 的骨架）
  // trigger_input 也纳入，但 executionOrder=0 且不计入进度分母
  const traces: NodeExecutionTrace[] = nodes.map((node, idx) => {
    const nodeData = node.data as AIStepNodeData;
    const meta = getNodeTypeMeta(node.type ?? 'chat_response');
    return {
      nodeId: node.id,
      nodeType: node.type ?? 'chat_response',
      nodeName: nodeData.label ?? meta.label,
      category: getNodeTheme(node.type ?? 'chat_response').category,
      status: 'pending',
      executionOrder: idx + 1,
      isParallel: false,
      streamingOutput: '',
      outputFormat: nodeData.output_format,
      modelRoute: nodeData.model_route,
    };
  });

  // 不计 trigger_input 的非 pending 节点数为分母
  const executableNodes = nodes.filter(n => n.type !== 'trigger_input');

  set({
    executionSession: {
      sessionId: crypto.randomUUID(),
      workflowId,
      startedAt: performance.now(),
      overallStatus: 'running',
      traces,
      completedCount: 0,
      totalCount: executableNodes.length,
    },
  });
},

registerNodeTrace: (nodeId, order, isParallel, parallelGroupId) => {
  set((state) => {
    if (!state.executionSession) return state;
    const traces = state.executionSession.traces.map(t =>
      t.nodeId === nodeId
        ? { ...t, executionOrder: order, isParallel, parallelGroupId, status: 'running' as NodeStatus }
        : t
    );
    return { executionSession: { ...state.executionSession, traces } };
  });
},

updateNodeTrace: (nodeId, updates) => {
  set((state) => {
    if (!state.executionSession) return state;
    let completedCount = state.executionSession.completedCount;
    const traces = state.executionSession.traces.map(t => {
      if (t.nodeId !== nodeId) return t;
      const updated = { ...t, ...updates };
      // 计算耗时
      if ((updates.status === 'done' || updates.status === 'error' || updates.status === 'skipped')
          && t.startedAt && !t.durationMs) {
        updated.durationMs = Math.round(performance.now() - t.startedAt);
        updated.finishedAt = performance.now();
        // 只对可执行节点计数（非 trigger_input）
        if (t.nodeType !== 'trigger_input') {
          completedCount = state.executionSession!.completedCount + 1;
        }
      }
      return updated;
    });
    return { executionSession: { ...state.executionSession, traces, completedCount } };
  });
},

appendNodeTraceToken: (nodeId, token) => {
  set((state) => {
    if (!state.executionSession) return state;
    const traces = state.executionSession.traces.map(t =>
      t.nodeId === nodeId ? { ...t, streamingOutput: t.streamingOutput + token } : t
    );
    return { executionSession: { ...state.executionSession, traces } };
  });
},

finalizeExecutionSession: (status) => {
  set((state) => {
    if (!state.executionSession) return state;
    const now = performance.now();
    return {
      executionSession: {
        ...state.executionSession,
        overallStatus: status,
        finishedAt: now,
        totalDurationMs: Math.round(now - state.executionSession.startedAt),
      },
    };
  });
},

clearExecutionSession: () => set({ executionSession: null }),
```

### 4.1 `use-workflow-execution.ts` 改造

**原则**：只在现有 SSE 处理器上叠加 trace 更新调用，不改变已有逻辑，向后兼容。

**改动要点**：

```typescript

  finalizeExecutionSession,
  clearExecutionSession,

  const id = workflowId ?? currentWorkflowId;
  if (!id) return;
  stop();
  setStatus('running');
  setError(null);
  clearExecutionSession();         // ← 新增：清除上次执行记录
  startExecutionSession(id);       // ← 新增：初始化新会话
  window.dispatchEvent(new Event('workflow:execution-start'));
  // ... 后续不变 ...
}, [...]);

// node_input 事件：新增 trace 注册
es.addEventListener('node_input', (e) => {
  try {
    const data = JSON.parse(e.data);
    startTimeMapRef.current[data.node_id] = performance.now();
    updateNodeData(data.node_id, { input_snapshot: data.input_snapshot });
    // ── 新增 ──
    updateNodeTrace(data.node_id, {
      status: 'running',
      startedAt: performance.now(),
      inputSummary: parseInputSummary(data.input_snapshot),
      rawInputSnapshot: data.input_snapshot,
    });
  } catch { /* ignore */ }
});

// node_token 事件：新增 trace token 追加
es.addEventListener('node_token', (e) => {
  try {
    const data = JSON.parse(e.data);
    setSelectedNodeId(data.node_id);
    updateNodeData(data.node_id, (prev) => ({ output: (prev.output ?? '') + data.token }));
    // ── 新增 ──
    appendNodeTraceToken(data.node_id, data.token);
  } catch { /* ignore */ }
});

// node_done 事件：finalize trace
es.addEventListener('node_done', (e) => {
  try {
    const data = JSON.parse(e.data);
    setSelectedNodeId(data.node_id);
    updateNodeData(data.node_id, { output: data.full_output, status: 'done' });
    // ── 新增 ──
    updateNodeTrace(data.node_id, {
      status: 'done',
      finalOutput: data.full_output,
    });
  } catch { /* ignore */ }
});

// node_status 事件：同步 trace 状态（skipped / error / waiting）
es.addEventListener('node_status', (e) => {
  try {
    const data = JSON.parse(e.data);
    // 已有逻辑...
    updateNodeData(data.node_id, updates);
    // ── 新增：同步非 running 状态到 trace ──
    if (data.status !== 'running') {
      updateNodeTrace(data.node_id, {
        status: data.status,
        ...(data.error ? { errorMessage: data.error } : {}),
      });
    }
  } catch { /* ignore */ }
});

// workflow_done 事件：finalize session
es.addEventListener('workflow_done', (e) => {
  try {
    const data = JSON.parse(e.data);
    setStatus(data.status === 'completed' ? 'completed' : 'error');
    // ── 新增 ──
    finalizeExecutionSession(data.status === 'completed' ? 'completed' : 'error');
  } catch {
    setStatus('completed');
    finalizeExecutionSession('completed');
  } finally {
    es.close();
    esRef.current = null;
    startTimeMapRef.current = {};
    window.dispatchEvent(new Event('workflow:execution-end'));
  }
});
```

### 5.5 `TraceStepInput.tsx` — 输入展示

```tsx

export const TraceStepInput: React.FC<TraceStepInputProps> = ({ trace, nodeNameMap }) => {

      )}

      {/* 上游输出：ID 转为节点名称 */}
      {Object.entries(parsed.upstream_outputs ?? {}).map(([nid, val]) => (
        <InputRow
          key={nid}
          label={nodeNameMap[nid] ?? `节点 ${nid.slice(0, 4)}`}  // ← 关键改造
          value={String(val)}
          maxChars={300}
          collapsible
        />
      ))}

      {/* node_config 参数（如有） */}
      {parsed.node_config && Object.keys(parsed.node_config).length > 0 && (
        <InputRow label="配置参数" value={JSON.stringify(parsed.node_config, null, 2)} />
      )}
    </div>
  );
};
```

### 5.6 `TraceStepOutput.tsx` — 输出展示（compact 渲染器规范）

**含 compact prop 的渲染器调用**：

```tsx
interface TraceStepOutputProps {
  trace: NodeExecutionTrace;
  compact: boolean;   // true = 推理链面板（精简模式），false = 完整展开
}

export const TraceStepOutput: React.FC<TraceStepOutputProps> = ({ trace, compact }) => {
  const Renderer = getRenderer(trace.nodeType);
  const output = trace.status === 'running' ? trace.streamingOutput : (trace.finalOutput ?? '');

  if (trace.errorMessage) {
    return <ErrorOutput message={trace.errorMessage} />;
  }

  return (
    <div>
      <Label>OUTPUT</Label>
      <Renderer
        output={output}
        format={trace.outputFormat ?? 'markdown'}
        nodeType={trace.nodeType}
        isStreaming={trace.status === 'running'}
        compact={compact}              // ← 新增 prop，传递给渲染器
      />
    </div>
  );
};
```

### 5.7 `TraceParallelGroup.tsx` — 并行节点组

当同批并行节点（`parallelGroupId` 相同）时，在时间轴上做特殊处理：

```
    ○ 步骤 3a — 大纲生成      ⟳ 并行执行
    ─┬──────────────────────────────────────
     │  ○ 步骤 3b — 内容提炼   ⟳ 并行执行
    ─┘  ──────────────────────────────────
```

内部结构：轴线从一点分叉为两条平行线，每条对应一个并行节点，完成后合并。
