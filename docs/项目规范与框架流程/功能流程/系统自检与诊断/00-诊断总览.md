# 00 — 系统诊断总览

## 1. 检测矩阵

| 类别 | 检测对象 | 检测方式 | 超时 | 并发控制 |
|------|----------|----------|------|----------|
| **数据库** | Supabase (PostgreSQL) | `SELECT 1` 查询 | 5s | 串行 |
| **AI 模型** | 所有 `catalog_skus.is_enabled=true` 的 SKU | 轻量 `chat.completions` (max_tokens=1) | 10s/个 | `Semaphore(5)` |
| **子 Agents** | `agents.yaml` 中 `enabled=true` 的 Agent | HTTP `GET /health/ready` | 5s/个 | `asyncio.gather` |
| **内部服务** | Embedding 服务 | 初始化探测 | 5s | 串行 |

**实现入口**：`backend/app/api/admin_diagnostics.py` → `DiagnosticsRunner`

---

## 2. 权限模型

| 触发面 | 认证方式 |
|--------|----------|
| Web UI `/admin-analysis/diagnostics` | `AdminJWTMiddleware` Cookie |
| HTTP API `GET /api/admin/diagnostics/full` | `admin_token` Cookie 或 `Authorization: Bearer` |
| PowerShell 脚本 | 环境变量 `STUDYSOLO_ADMIN_TOKEN` 或参数 `-AdminToken` |
| AI Skill | 走 PowerShell 脚本，继承环境变量 |

**脱敏策略**：后端对 `error` 字段过滤 `api_key` 关键字，避免日志泄露密钥。

## 4. 日志约定

| 位置 | 命名规则 | 编码 |
|------|----------|------|
| 日志主文件 | `scripts/logs/diagnostics-YYYYMMDD-HHmmss.log` | UTF-8 |
| Markdown 报告 | `scripts/logs/diagnostics-YYYYMMDD-HHmmss.md` | UTF-8 |
| JSON 报告 | `scripts/logs/diagnostics-YYYYMMDD-HHmmss.json` | UTF-8 |
| 纯文本报告 | `scripts/logs/diagnostics-YYYYMMDD-HHmmss.txt` | UTF-8 |

- `scripts/logs/` 目录已在 `.gitignore` 中
- 建议保留最近 30 天
- 敏感信息已在后端脱敏，但仍建议不要跨团队共享原始日志

---

## 5. 退出码语义（CLI）

| 退出码 | 含义 |
|--------|------|
| `0` | 所有组件 healthy |
| `1` | 存在 unhealthy 组件（业务异常） |
| `2` | 脚本自身异常（网络失败、鉴权失败、后端未启动） |

CI 可直接使用退出码做健康门禁。
