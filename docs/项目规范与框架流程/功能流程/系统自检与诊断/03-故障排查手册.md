# 03 — 故障排查手册

## 快速查找表

| 现象 | 类别 | 可能原因 | 排查步骤 |
|------|------|----------|----------|
| `database` 组件 `Connection timeout` | DB | Supabase 宕机 / 网络隔离 / URL 错误 | 1. 检查 `backend/.env` 的 `SUPABASE_URL`<br>2. `curl <SUPABASE_URL>`<br>3. 看 Supabase 控制台 |
| `ai_model` 组件 `401 Unauthorized` | AI | API Key 失效 / 被撤销 | 1. 到对应 Provider 控制台验证 Key<br>2. 重新设置环境变量<br>3. 重启后端 |
| `ai_model` 组件 `Rate limit exceeded` | AI | 触发限流 | 1. 稍后重试<br>2. 升级 Provider 配额<br>3. 调整 catalog 路由策略 |
| `agent` 组件 `Connection refused` | Agent | 子 Agent 服务未启动 | 1. 确认 Agent 进程：`Get-Process python`<br>2. 检查 `agents/<name>/README.md` 启动指令<br>3. 查 Agent 日志 |
| `agent` 组件 `Timeout after 5000ms` | Agent | Agent 响应慢 / 依赖卡住 | 1. 查 Agent 自身日志<br>2. 测试 Agent 依赖的外部 API<br>3. 提高超时（临时） |
| `service` 组件 embedding 失败 | Service | 模型权重缺失 / 显存不足 | 1. 看后端启动日志<br>2. 验证 embedding 配置<br>3. 重启后端 |
| 脚本退出码 2 + `Backend not reachable` | 脚本 | 后端未启动 | 运行 `./scripts/start-studysolo.ps1` |
| 脚本退出码 2 + `401 Unauthorized` | 脚本 | Admin Token 失效 | 重新登录 `/admin-analysis/login` 获取新 Token |

## 通用排查流程

### Step 1：看报告摘要

打开最新的 `scripts/logs/diagnostics-*.md`：

```markdown
## Summary
- Total: 15
- Healthy: 13
- Unhealthy: 2

## Unhealthy Components
### ai-zhipu-glm-4-plus (ai_model)
- Error: Connection timeout after 10000ms
- Latency: 10000ms
```

### Step 2：定位到具体组件

`.json` 报告的 `components[i].details` 通常包含更多上下文（如 `provider`、`url`、`sku_id`）。

### Step 3：查后端日志

```powershell
Get-Content backend\logs\backend.log -Tail 100
```

重点关注与组件 ID 同时间段的错误堆栈。

### Step 4：本地复现

对单个 AI 模型：

```python
# 在 backend/ 目录
python -c "
from app.services.llm.provider import get_client
import asyncio
async def test():
    client = get_client('openai')
    r = await client.chat.completions.create(
        model='gpt-4o', messages=[{'role':'user','content':'hi'}], max_tokens=1
    )
    print(r)
asyncio.run(test())
"
```

对单个 Agent：

```powershell
curl http://127.0.0.1:8001/health/ready
```

### Step 5：修复后回归

```powershell
.\scripts\diagnostics\run-diagnostics.ps1 -Category <修复的类别>
```

## 常见误报

| 误报 | 原因 | 处置 |
|------|------|------|
| 首次调用某个冷启动模型超时 | Provider 冷启动延迟 | 重跑一次 |
| 本地开发 Agents 未全部启动 | 开发时通常只启动一两个 Agent | 禁用 `agents.yaml` 中未启动项的 `enabled` |
| 网络波动导致单次失败 | 瞬时抖动 | 重跑；连续 3 次失败再排查 |
