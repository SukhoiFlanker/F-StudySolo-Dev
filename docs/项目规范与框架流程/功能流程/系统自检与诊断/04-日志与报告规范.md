# 04 — 日志与报告规范

## 1. 存储位置

**所有诊断日志统一落盘** `scripts/logs/`。

该目录：
- 已通过 `.gitignore` 排除内容
- 保留 `.gitkeep` 维持目录结构
- 不会被 git 跟踪

---

## 2. 命名规则

```text
scripts/logs/diagnostics-{YYYYMMDD}-{HHmmss}.{ext}
```

| 扩展名 | 用途 | 必产 |
|--------|------|------|
| `.log` | 主日志：执行过程 + 摘要 | ✅ 每次必产 |
| `.md` | Markdown 报告：可粘贴到 PR/Issue | 默认产 |
| `.json` | 机器可读：CI/脚本解析 | 默认产 |
| `.txt` | 纯文本报告：复制到即时通讯工具 | 默认产 |

**示例**：

```text
scripts/logs/
├── .gitignore
├── .gitkeep
├── diagnostics-20260417-104732.log
├── diagnostics-20260417-104732.md
├── diagnostics-20260417-104732.json
└── diagnostics-20260417-104732.txt
```

## 4. 保留策略

| 环境 | 策略 |
|------|------|
| 本地开发 | 建议 30 天自动清理 |
| CI | 构建产物形式上传，本地不留存 |
| 生产运维 | 按需手动归档，不长期保留 |

**手动清理**：

```powershell
# 清理 30 天前的诊断日志
Get-ChildItem scripts\logs\diagnostics-*.* |
    Where-Object { $_.LastWriteTime -lt (Get-Date).AddDays(-30) } |
    Remove-Item
```

## 5. 脱敏规则

后端 `backend/app/api/admin_diagnostics.py` 的 `_sanitize_error` 已做：

- 过滤包含 `api_key` 关键字的消息
- 替换为 `Authentication failed` 等通用描述

**仍需人工注意**：

- URL 中可能含用户名 / 端口信息
- Provider 名称、模型 ID 属于业务信息，可接受
- 若发现敏感泄露，立即：
  1. 删除相关日志文件
  2. 向安全负责人报告
  3. 在 `admin_diagnostics.py` 补脱敏规则

## 6. 脚本编码约定

| 文件类型 | 编码 | 行尾 |
|----------|------|------|
| `*.ps1` (PowerShell) | **UTF-8 with BOM**（必需） | CRLF |
| `*.sh` (Shell) | UTF-8 无 BOM | LF |
| `*.py` (Python) | UTF-8 无 BOM | LF |
| `*.md` (文档) | UTF-8 无 BOM | LF |
| 日志输出 | UTF-8 无 BOM | 任意 |

**PowerShell 必须 BOM 的原因**：Windows PowerShell 5.1 在没有 BOM 时默认按 GBK 解析，中文会乱码。PS 7+ 虽然默认 UTF-8，但加 BOM 可同时兼容两版本。

**验证 BOM**：

```powershell
$bytes = [System.IO.File]::ReadAllBytes("scripts\diagnostics\run-diagnostics.ps1")
"{0:X2} {1:X2} {2:X2}" -f $bytes[0], $bytes[1], $bytes[2]
# 期望输出: EF BB BF
```
