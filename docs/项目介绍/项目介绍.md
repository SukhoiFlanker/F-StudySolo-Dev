## 第一章 · 产品概述

### 1.1 产品定位

**StudySolo** 是 1037Solo 生态下的核心次时代子项目，定位为 **新一代 AI 驱动的可视化学习工作流平台**。

如果说 1037Solo 主平台是用户获取信息与进行碎片化对话的入口，那么 StudySolo 则是用户 **将长线目标具象化、将复杂学习过程拆解并自动化深度执行** 的"第二大脑操作系统"。

它创新性地将大语言模型（LLM）的分析规划能力与节点式（Node-based）可视化画布相结合。用户只需输入一个宽泛的课题目标，系统便会自动规划出包含"分析 → 分解 → 检索 → 结构化生成"等环节的定制化专属工作流，并利用内置的多模型引擎进行流式执行，带来沉浸式、极客范的高效知识链路体验。

### 1.2 我们究竟在解决什么问题？

| 痛点 | 传统方案的困境 | StudySolo 的解法 |
|------|--------------|-----------------|
| **学习路径的"重度迷雾"** | 面对崭新宏大的交叉领域课题，传统聊天式 AI 会陷入"查到深处忘出来"的窘境 | 抛弃问答轴，直接生成类似心智图的网格化学习链路，一目了然 |
| **割裂的 AI 工具流水线** | 深度学习需要提取大纲、扩写、跨语种翻译、检索多库资料、制作闪卡……往往需要不停地复制文本切换窗口 | 所有环节化为连在同一根管子上的组件节点，一键贯穿全流程 |
| **黑箱化不可控的焦虑** | 常规 AI 黑盒式地吐出结果，一旦跑偏用户无法半路干预修订上下文 | 在节点画布上，任何一个出错的上游生成都可以被随时暂停、修改，然后继续传导 |

### 1.3 产品愿景

> **让每一个人都能拥有一套属于自己的、可视化的、AI 驱动的知识生产流水线。**

StudySolo 致力于打造一个"所想即所得"的知识引擎——你只需要说出你想学什么，剩下的拆解、规划、检索、生成、总结、出题、导出，全部由系统在一块透明可控的画布上自动完成。

## 第二章 · 平台架构总览

StudySolo 采用前后端分离的现代化企业级架构，不仅完全继承而且突破了 1037Solo 生态的技术标准。

### 2.1 前端技术栈

| 技术 | 版本 | 职责 |
|------|------|------|
| **Next.js** | 16 | App Router 架构，服务端渲染与路由控制 |
| **React** | 19 | UI 组件化渲染引擎 |
| **Tailwind CSS** | v4 | 高质感暗色主题设计系统与全景色彩变盘 |
| **@xyflow/react** | 最新版 | 硬件级渲染的工作流可视化拓扑画布引擎 |
| **Zustand** | 最新版 | 极低延迟的全局动态画布状态管理 |

前端以极其严苛的工程规范构建（单文件 ≤ 300 行，入口页 ≤ 220 行），严禁组件内夹杂异步网络逻辑，所有数据操作统一委托至 Hooks 与 Services 层。

### 2.2 后端技术栈

| 技术 | 版本 | 职责 |
|------|------|------|
| **FastAPI** | 最新版 | 全异步 ASGI 弹性架构，承载 API 端点与 SSE 推流 |
| **工作流拓扑编排引擎** | 自研 | 基于 Kahn 算法的 DAG 拓扑排序 + 分层并行执行 |
| **动态节点插件系统** | 自研 | 基于 `__init_subclass__` 的零配置自动注册节点体系 |
| **AI 智能路由** | 自研 | 多模型多平台动态调度 + 秒级容灾链路切换 |

后端核心亮点在于 **插件化节点架构**：所有节点类型（分析、生成、检索、导出等）均部署为完全解耦的独立微插件，继承统一的 `BaseNode` 基类，新增一个节点功能只需 3 步即可集成上线。

### 2.3 数据库底座

- **平台**：Supabase（与 1037Solo 共享同一个 Project）
- **认证**：复用 Supabase Native Auth，达成 One-ID JWT 联合登录
- **安全**：采用严格的行级安全策略（Row Level Security），实现物理级别的租户数据隔离
- **实时**：支持 Supabase Realtime 订阅，画布状态变更可即时推送

### 2.4 多模型 AI 引擎矩阵

StudySolo 同时接入了 **8 大 AI 平台**，组成强大的多模型引擎矩阵：

| **DeepSeek** | DeepSeek-R1 / DeepSeek-V3 | 深度推理、逻辑分析、学术级思考 |
| **月之暗面（Moonshot）** | Moonshot-v1-8k | 长文本理解与生成 |
| **火山引擎** | 豆包 Pro-32k | 超长上下文窗口处理 |

### 2.5 智能路由与容灾机制

系统将模型划分为两条能力链路，并配备完整的降级策略：

**Chain A — 格式严格链**（用于需要精确 JSON 输出的场景）：
```
DashScope Qwen3-Turbo → DeepSeek-V3 → Moonshot-v1-8k
```

**Chain B — 深度推理链**（用于需要强化思考能力的场景）：
```
DeepSeek-R1 → DashScope Qwen3-Plus → 火山引擎 Doubao-Pro-32k
```

- **超时保护**：每次请求 30 秒超时自动切换
- **最大重试**：3 次智能降级后触发最终兜底
- **代理平台隔离**：格式严格链（Chain A）禁止使用代理聚合平台，确保输出格式精准

## 第三章 · 核心能力

### 3.1 动态生成式工作流

您只需输入一句话描述您的学习目标，例如："我想在三十天内入门流体力学并能看懂基础方程"。系统会经历以下自动流程：

1. **AI 分析器** 解读您的自然语言需求，提取核心目标、步骤、约束条件
2. **AI 规划器** 基于分析结果，在画布上自动绘制出由数十个节点组成的攻坚拓扑图
3. 所有节点之间自动建立依赖连线，形成完整的有向无环图（DAG）

整个过程无需任何手动操作，从空白画布到完整工作流，仅需数秒。

### 3.2 可视化交互画布

基于硬件级渲染引擎 `@xyflow/react`，StudySolo 将所有学习路径变为可物理操作的实体控件。您可以：

- 🖱️ **自由拖拽** 任意节点，重新组织学习顺序
- ✂️ **剪断连线** 移除不需要的步骤
- 🔗 **拉出分支** 将某个节点的产出链接到多个下游（如同时分流给"翻译"和"出题"）
- 🗑️ **删除废节点** 去掉 AI 过度规划的冗余环节
- ✏️ **修改标签** 精细调整每个节点的任务描述

### 3.3 插件化节点生态

系统的所有功能均以 **完全解耦的独立插件** 形式存在。每个节点就像乐高积木——独立封装、互不干扰、即插即用。目前已内置 **5 大类 18 种功能节点**（详见[第四章](#第四章--节点功能大全)）。

新增一个节点能力仅需 3 步：
1. 在后端创建节点目录，编写执行逻辑与提示词
2. 在配置文件中注册模型路由
3. 在前端挂载渲染视窗

### 3.4 精准上下文穿透

不同于传统系统粗暴地把所有对话历史合并成一条长串，StudySolo 使用 **空间几何式的关联传输**：

- 每个节点的上下文记忆 **物理上只来源于与它连通的线引流下来的数据**
- 引擎沿着 Edge 有向边精准抽取上游输出并注入下游
- 极大避免了大模型的幻觉污染和无关信息干扰

### 3.5 拓扑流式执行引擎

当您按下"运行"按钮后，引擎会：

1. 对所有节点进行 **拓扑排序**（Kahn 算法），确保按依赖顺序执行
2. **同一层级的独立节点并行执行**，最大化利用算力
3. 每个节点的 AI 生成过程以 **SSE（Server-Sent Events）** 实时流式推送到前端
4. 您可以像看工厂流水线一样，实时观看数据从上游"流入"下游

如果某个节点执行失败，引擎会自动：
- 将该节点标记为红色错误态
- 阻断所有下游节点（防止污染扩散）
- 支持单独对失败节点"重跑"

### 3.6 三级防抖云端同步

为了在海量节点拖拽操作下仍保持 120 帧极致流畅体验，StudySolo 采用独创的三级降压落盘策略：

```
UI层 (0ms 无感知) → IndexedDB (500ms 本地暂存) → Supabase (4s 云端静默同步)
```

- **第一级**：画布交互即时响应，零延迟
- **第二级**：变更暂存至浏览器 IndexedDB，防止频繁网络请求
- **第三级**：检测用户停止操作后，静默与云端 Supabase 完成数据同步

即便遭遇强制关机，IndexedDB 中的备份快照也能最大程度恢复状态。

### 3.7 One-ID 跨平台互通

StudySolo 与 1037Solo 主平台共享同一身份体系：

- 🔑 **单点登录**：一次登录，双端通行
- 💰 **资产共享**：配额、积分、订阅层级全生命周期通用
- 🔄 **无缝切流**：可在两个平台间随意调度资产

### 3.8 全景管理员控制台

内建庞大且细致的管理员后台，包含十大管理维度：

| 管理模块 | 功能说明 |

| **用户管理** | 用户列表、角色权限分配、账户状态管控 |
| **工作流管理** | 全平台工作流审计、运行状态实时监控 |
| **模型配置** | AI 模型路由配置、平台密钥管理 |
| **审计日志** | 操作追踪、安全事件记录 |
| **评价管理** | 用户反馈收集与分析 |
| **成员管理** | 团队成员权限管理 |
| **公告管理** | 系统公告发布与维护 |

| **系统配置** | 全局参数调整、功能开关 |

## 第四章 · 节点功能大全

### 4.1 节点体系概述

StudySolo 的节点体系分为 **5 大类别**，共计 **18 种功能节点**。每个节点都是一个独立的微型专家，各司其职，通过连线协同工作。

```
🟢 输入类 (Input)          → 数据从这里进入工作流
🟣 分析类 (Analysis)       → 对数据进行理解、拆解、决策
🔵 生成类 (Generation)     → 根据分析结果产出知识内容
🩷 交互类 (Interaction)    → 与用户进行个性化对话
⚫ 输出类 (Output)          → 将结果保存或导出
```

### 4.2 输入类节点 (Input)

输入类节点是工作流的"大门"——所有的数据和知识都从这里进入系统。

---

#### ▶️ 触发输入节点 · Trigger Input

| 属性 | 说明 |
|------|------|
| **节点代号** | `trigger_input` |
| **节点颜色** | 🟢 翡翠绿 `#10b981` |
| **是否调用 AI** | ❌ 否 |
| **输出格式** | 原样透传 |

**功能介绍**

触发输入节点是每一个工作流的 **起点**。它的工作很简单但极其关键：接收您在画布顶部输入框中键入的学习目标原文，并将其原封不动地传递给下游节点。

**它能为您做什么？**

当您在 StudySolo 的输入栏中写下"我想系统学习 Python 数据分析，从 Pandas 入门到实战项目"这样的话，触发输入节点会：
1. 捕获您的完整输入文本
2. 作为工作流的第一个节点启动执行
3. 将您的原始需求精确传递给下游的 AI 分析器等节点

**适用场景**
- 任何工作流的必备入口
- 接收用户的初始学习目标或任务描述

#### 📚 知识库检索节点 · Knowledge Base

| 属性 | 说明 |
|------|------|
| **节点代号** | `knowledge_base` |
| **节点颜色** | 🟣 深紫 `#8b5cf6` |
| **是否调用 AI** | ❌ 否（调用检索服务） |
| **输出格式** | Markdown |

**功能介绍**

知识库检索节点是您的 **私人图书馆管理员**。它不调用 AI 大模型，而是直接在您预先上传的文档库中进行语义检索，帮您找到与当前学习主题最相关的资料片段。

**它能为您做什么？**

假设您之前上传了多本机器学习教材的 PDF，当工作流运行到知识库节点时，它会：
1. 根据当前任务关键词（来自上游节点），在您的文档库中进行智能搜索
2. 按相关性排序返回最匹配的 **Top-K 段落**（默认 5 条，相似度阈值 0.7）
3. 每条结果附带来源引用，方便您回溯原文

**可配置参数**
- `top_k`：返回结果数量（默认 5 条）
- `threshold`：相似度阈值（默认 0.7，越高越精准）

**适用场景**
- 基于私有文档进行深度学习
- 需要从之前积累的教材、笔记中提取知识
- 与内容提炼节点配合，先检索再精炼

**典型连线**
```
[触发输入] → [知识库检索] → [内容提炼] → [总结]
```

#### 🌐 联网搜索节点 · Web Search

| 属性 | 说明 |
|------|------|
| **节点代号** | `web_search` |
| **节点颜色** | 🔵 天空蓝 `#0ea5e9` |
| **是否调用 AI** | ❌ 否（调用 Tavily 搜索 API） |
| **输出格式** | Markdown |

**功能介绍**

联网搜索节点是您的 **互联网信息侦察兵**。当您需要学习最新的技术、时事或任何互联网上的实时知识时，这个节点会帮您从全网搜索并整理出结构化的参考资料。

**它能为您做什么？**

当工作流运行到联网搜索节点时，它会：
1. 根据节点标签和上游输出自动构建精准的搜索关键词
2. 通过 Tavily 搜索引擎 API 检索互联网最新内容
3. 返回包含 **AI 摘要 + 逐条结果（标题、来源、内容）** 的格式化 Markdown

**可配置参数**
- `max_results`：最大返回条数（默认 5 条）
- `search_depth`：搜索深度，`basic`（快速）或 `advanced`（深度）

**适用场景**
- 学习最前沿的技术或学术成果
- 需要实时数据辅助学习（如最新论文、行业报告）
- 与内容提炼节点配合做深度分析

**典型连线**
```
[触发输入] → [联网搜索] → [内容提炼] → [闪卡] / [总结]
```

**它能为您做什么？**
