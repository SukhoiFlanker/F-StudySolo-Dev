# 分阶段落地路线（P0 到 P2）

## P0（先上线可用主链路）

目标：跑通 `输入 -> 分析 -> 规划 -> 执行 -> 流式反馈 -> 入库`

- 完成节点状态机与拓扑执行
- 完成 `ai_analyzer/ai_planner` 结构化输出校验
- 完成 SSE 输出与基础运行看板
- 完成基础配额（日执行数/并发）

验收：
- 典型学习任务可稳定执行，失败可追踪。

## P1（能力增强）

目标：实现你规划中的“RAG + 联网 + 人在回路”

- 上线知识库三层漏斗 + 混合检索
- 上线暂停/人工覆写/恢复
- 引入 Celery + Redis 处理重任务
- 上线导出（PDF/DOCX）

验收：
- 复杂任务可人工纠偏继续跑，知识库回答可溯源。

## P2（规模化与商业化）

目标：提高稳定性、可观测性与付费转化效率

- 完整会员/加购账本与风控
- 高级缓存策略与成本看板
- 高级节点（loop_map、logic_switch、code_script）
- 更细粒度 RLS 与团队空间能力

验收：
- 可支撑多用户并发，单位问答成本下降，会员权益边界清晰。

## 推荐启动顺序（务实版）

1. 先做 P0 的“结构化输出稳定性 + 状态机正确性”。
2. 再做 P1 的“混合检索 + 人工覆写”。
3. 最后做 P2 的“计费精细化与运营能力”。
