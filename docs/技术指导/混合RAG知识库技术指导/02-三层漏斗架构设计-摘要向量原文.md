# 三层漏斗架构设计（摘要-向量-原文）

## 1. 目标

在保证答案质量前提下，把“每问必读全文”改为“先摘要、后向量、再原文”，控制 token 成本与延迟。

## 2. 分层定义

1. 摘要层（第一优先）
- 存：文档摘要、核心观点、章节结构、关键术语。
- 用：覆盖 60%~80%常见问答。

2. 向量层（第二优先）
- 存：摘要向量 + chunk 向量（可分两张表）。
- 用：检索 3~8 个高相关片段补充上下文。

3. 原文层（第三优先）
- 存：对象存储中的原文件与可定位片段。
- 用：精确引用、审计溯源、用户点击“查看原文”。

## 3. 文档预处理建议

- 分块策略：优先 `by_title`，并设置 overlap。
- 经验参数：
  - chunk 长度：500~800 tokens
  - overlap：80~120 tokens
- 解析失败兜底：
  - PDF OCR失败时写入失败队列，异步重试。

## 4. 质量约束

- chunk 入库前去重（hash）。
- 同一文档保留版本号（便于重建向量）。
- 摘要生成失败不阻塞原文入库：允许稍后补摘要。
