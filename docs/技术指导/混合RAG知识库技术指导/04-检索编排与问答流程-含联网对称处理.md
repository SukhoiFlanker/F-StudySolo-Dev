# 检索编排与问答流程（含联网对称处理）

## 1. 本地知识库问答流程

1. Query 预处理
- 问题归一化（去冗词、识别意图：概览/细节/引用）。

2. 第一跳：摘要检索
- 在 `kb_summary_embeddings` 召回 topK=3。
- 若可回答，直接生成并返回来源。

3. 第二跳：chunk 深检索
- 在 `kb_chunk_embeddings` 召回 topK=8。
- 可选重排（rerank）后截断到 topN=4。

4. 第三跳：原文定位
- 基于 `document_id + section/page/offset` 返回原文片段。

## 2. 联网搜索对称流程

1. 多源检索（权威源 + 社区源）
2. 先摘要化再问答（避免直接塞整页）
3. 追问/证据要求时再展开网页原文

## 3. 输出格式建议

- `answer`
- `citations[]`（doc_id/url + chunk_id + snippet）
- `confidence`
- `route`（summary_only / summary+chunk / raw_source）

## 4. 失败处理

- 召回为空：提示用户缩小范围或指定文档。
- 模型超时：自动降级到较小模型或减少上下文。
- 结果冲突：返回多来源并标注分歧。
