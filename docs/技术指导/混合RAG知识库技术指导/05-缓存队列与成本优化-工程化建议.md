# 缓存、队列与成本优化（工程化建议）

## 1. 缓存策略

1. 答案缓存（强烈建议）
- key：`hash(user_scope + normalized_query + retrieval_version)`
- TTL：10分钟~24小时（按业务热度）

2. 检索缓存
- 缓存 query embedding 与 topK 召回结果。

3. 失效机制
- 文档更新后，按 `document_id/version` 精确失效。

## 2. 异步任务策略

### MVP
- 使用 FastAPI `BackgroundTasks` 足够。
- 用于：上传后摘要生成、分块向量化、重试。

### 成长期
- 引入 Celery/RQ + Redis 队列。
- 适合：高并发、长任务、可观测重试。

## 3. 成本控制

- 先摘要再深检索：优先级最高。
- 限制上下文预算：控制 max_context_tokens。
- 对“高频问题”做缓存 + 预计算。
- 仅对 topN 候选做 rerank。

## 4. 关键监控指标

- 检索：Recall@K、MRR、HitRate
- 生成：平均 token、平均耗时、失败率
- 成本：单问答成本、缓存命中率
- 体验：首 token 时间（SSE）与完答时延
