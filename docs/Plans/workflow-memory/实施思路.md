## 一、整体架构设计

### 1.1 系统分层

```
┌─────────────────────────────────────────────────────┐
│                    前端展示层                         │
│  ┌──────────┐  ┌──────────┐  ┌───────────────────┐  │
│  │ m/[id]   │  │ 运行日志  │  │ 知识库"所有记录"  │  │
│  │ 记忆页面  │  │ 弹出列表  │  │ Tab              │  │
│  └──────────┘  └──────────┘  └───────────────────┘  │
├─────────────────────────────────────────────────────┤
│                    API 层                            │
│  GET /api/workflow/{id}/runs      → 运行列表         │
│  GET /api/workflow/run/{runId}    → 运行详情+traces  │
│  POST /api/workflow/run/{id}/share → 切换分享状态    │
│  GET /api/workflow/run/{id}/public → 公开访问        │
│  GET /api/knowledge/all-records   → 所有运行记录     │
├─────────────────────────────────────────────────────┤
│                    数据层                            │
│  ss_workflow_runs (扩展 is_shared/share_token)       │
│  ss_workflow_run_traces (新表：节点级执行记录)        │
├─────────────────────────────────────────────────────┤
│                    执行引擎层                         │
│  workflow_execute.py → 执行时累积 traces → 完成后写入 │
└─────────────────────────────────────────────────────┘
```

### 1.2 数据流

```
工作流执行
  → SSE 事件流（node_input / node_token / node_done / node_status）
  → 后端 event_generator 中累积 node_traces dict
  → workflow_done 后 _finalize_run 批量写入 ss_workflow_run_traces
  → 前端可随时通过 API 查询历史运行的完整 traces
  → m/[id] 页面渲染节点级纸条卡片
```

## 三、后端实施方案

### 3.1 执行引擎改造（workflow_execute.py）

核心改动：在 `event_generator()` 中累积节点执行数据，完成后批量写入。

```python
# workflow_execute.py — event_generator 内部

node_traces: dict[str, dict] = {}
trace_order = 0

async for event in execute_workflow(...):
    yield event
    try:
        evt = _parse_sse_data(event)
        if not evt:
            continue
        event_type = event.split("\n")[0].replace("event: ", "")

        if event_type == "node_input":
            trace_order += 1
            node_traces[evt["node_id"]] = {
                "node_id": evt["node_id"],
                "execution_order": trace_order,
                "input_snapshot": evt.get("input_snapshot"),
                "status": "running",
            }

        elif event_type == "node_status":
            nid = evt["node_id"]
            if nid in node_traces:
                node_traces[nid]["status"] = evt["status"]
                if evt.get("error"):
                    node_traces[nid]["error_message"] = evt["error"]

        elif event_type == "node_done":
            nid = evt["node_id"]
            if nid in node_traces:
                node_traces[nid]["final_output"] = evt.get("full_output")
                node_traces[nid]["status"] = "done"

    except Exception:
        pass

# 在 _finalize_run 之后调用
if workflow_run_ref and node_traces:
    await _save_traces(service_db, run_id, nodes, node_traces)
```

```python
async def _save_traces(
    db: AsyncClient,
    run_id: str,
    nodes: list[dict],
    node_traces: dict[str, dict],
) -> None:
    """批量写入节点级执行记录。"""
    node_map = {n["id"]: n for n in nodes}
    rows = []
    for nid, trace in node_traces.items():
        node_def = node_map.get(nid, {})
        node_data = node_def.get("data", {})
        rows.append({
            "run_id": run_id,
            "node_id": nid,
            "node_type": node_def.get("type", "unknown"),
            "node_name": node_data.get("label", nid),
            "category": _get_category(node_def.get("type")),
            "execution_order": trace.get("execution_order", 0),
            "status": trace.get("status", "unknown"),
            "input_snapshot": trace.get("input_snapshot"),
            "final_output": trace.get("final_output"),
            "output_format": node_data.get("output_format"),
            "duration_ms": trace.get("duration_ms"),
            "model_route": node_data.get("model_route"),
            "is_parallel": trace.get("is_parallel", False),
            "parallel_group_id": trace.get("parallel_group_id"),
            "error_message": trace.get("error_message"),
        })
    if rows:
        try:
            await db.from_("ss_workflow_run_traces").insert(rows).execute()
        except Exception as e:
            logger.error("Failed to save run traces for %s: %s", run_id, e)
```

### 3.2 新增 API 端点

**运行列表 — GET /api/workflow/{workflow_id}/runs**

```python
@router.get("/{workflow_id}/runs")
async def get_workflow_runs(
    workflow_id: str,
    limit: int = 20,
    offset: int = 0,
    current_user: dict = Depends(get_current_user),
    db: AsyncClient = Depends(get_supabase_client),
):
    """获取某个工作流的运行历史列表。"""
    result = await db.from_("ss_workflow_runs") \
        .select("id, status, input, tokens_used, started_at, completed_at, is_shared") \
        .eq("workflow_id", workflow_id) \
        .eq("user_id", current_user["id"]) \
        .order("started_at", desc=True) \
        .range(offset, offset + limit - 1) \
        .execute()
    return result.data or []
```

**运行详情 + traces — GET /api/workflow/run/{run_id}**

```python
@router.get("/run/{run_id}")
async def get_run_detail(
    run_id: str,
    current_user: dict = Depends(get_current_user),
    db: AsyncClient = Depends(get_supabase_client),
):
    """获取某次运行的详情 + 所有节点 traces。"""
    run = await db.from_("ss_workflow_runs") \
        .select("*") \
        .eq("id", run_id) \
        .eq("user_id", current_user["id"]) \
        .single() \
        .execute()
    if not run.data:
        raise HTTPException(404, "运行记录不存在")

    traces = await db.from_("ss_workflow_run_traces") \
        .select("*") \
        .eq("run_id", run_id) \
        .order("execution_order") \
        .execute()

    return {**run.data, "traces": traces.data or []}
```

**切换分享 — POST /api/workflow/run/{run_id}/share**

```python
@router.post("/run/{run_id}/share")
async def toggle_run_share(
    run_id: str,
    current_user: dict = Depends(get_current_user),
    db: AsyncClient = Depends(get_supabase_client),
):
    """切换运行记录的分享状态。"""
    run = await db.from_("ss_workflow_runs") \
        .select("id, is_shared, share_token") \
        .eq("id", run_id) \
        .eq("user_id", current_user["id"]) \
        .single() \
        .execute()
    if not run.data:
        raise HTTPException(404)

    new_shared = not run.data.get("is_shared", False)
    update = {"is_shared": new_shared, "shared_at": datetime.now(timezone.utc).isoformat() if new_shared else None}
    if new_shared and not run.data.get("share_token"):
        update["share_token"] = str(uuid.uuid4())[:8]  # 短 token

    await db.from_("ss_workflow_runs").update(update).eq("id", run_id).execute()
    return {"is_shared": new_shared, "share_token": update.get("share_token") or run.data.get("share_token")}
```

**公开访问 — GET /api/workflow/run/{run_id}/public**

```python
@router.get("/run/{run_id}/public")
async def get_public_run(
    run_id: str,
    db: AsyncClient = Depends(get_db),  # service_db，不需要用户鉴权
):
    """公开访问已分享的运行记录。"""
    run = await db.from_("ss_workflow_runs") \
        .select("id, workflow_id, input, status, tokens_used, started_at, completed_at") \
        .eq("id", run_id) \
        .eq("is_shared", True) \
        .single() \
        .execute()
    if not run.data:
        raise HTTPException(404)

    # 获取工作流名称
    wf = await db.from_("ss_workflows") \
        .select("name") \
        .eq("id", run.data["workflow_id"]) \
        .single() \
        .execute()

    traces = await db.from_("ss_workflow_run_traces") \
        .select("*") \
        .eq("run_id", run_id) \
        .order("execution_order") \
        .execute()

    return {
        **run.data,
        "workflow_name": wf.data["name"] if wf.data else "未知工作流",
        "traces": traces.data or [],
    }
```

### 4.4 分享机制

**分享开关**

- 仅运行记录所有者可见
- 默认关闭（`is_shared = false`）
- 点击开启后生成分享链接：`{origin}/m/{runId}`
- 分享链接可复制到剪贴板
- 再次点击可关闭分享

**访问控制逻辑**

```
访问 /m/{runId}
  → Server Component 尝试获取运行详情
  → 情况 1：is_shared = true → 任何人可访问（调用 public API）
  → 情况 2：is_shared = false + 已登录 + 是所有者 → 可访问（调用 authed API）
  → 情况 3：is_shared = false + 非所有者 → 404
```

## 六、技术决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| traces 存储方式 | 独立表（非 JSONB） | 可索引、可分页、可按节点类型查询 |
| traces 写入时机 | 后端执行完成后批量写入 | 后端已有 SSE 事件数据，无需前端上报 |
| 分享粒度 | 运行级别（非工作流级别） | 用户可能只想分享某次运行结果 |
| 分享链接格式 | `/m/{runId}` | 简洁，与 `/s/{workflowId}` 对称 |
| 访问控制 | `is_shared` 字段 + RLS | 与现有 `is_public` 模式一致 |
| m/[id] 布局 | 复用 s/[id] 的独立布局模式 | 减少重复代码，保持一致体验 |
| 节点卡片渲染 | 复用 TraceStepItem 的逻辑 | 已有 Markdown 渲染和状态展示 |

## 七、风险与注意事项

1. **数据量**：每次运行可能产生 10-20 条 traces，每条含完整输出文本。需要考虑：
   - `final_output` 字段使用 TEXT 类型（不限长度）
   - 列表查询时不返回 `final_output`，仅在详情页加载
   - 考虑对超过 30 天的 traces 进行归档或压缩

2. **执行引擎性能**：在 `event_generator` 中累积 traces 是内存操作，影响极小。批量写入在执行完成后进行，不影响 SSE 流。

3. **RLS 策略复杂度**：`ss_workflow_run_traces` 的 RLS 通过子查询关联 `ss_workflow_runs.user_id`，可能有性能影响。可以考虑在 traces 表中冗余 `user_id` 字段。

4. **向后兼容**：已有的 `ss_workflow_runs` 记录没有 traces 数据。m/[id] 页面需要处理 traces 为空的情况（显示"该运行记录暂无详细节点数据"）。

5. **分享安全**：`share_token` 使用 UUID 前 8 位，碰撞概率极低但非零。如果安全要求更高，可以使用完整 UUID 或加密 token。
