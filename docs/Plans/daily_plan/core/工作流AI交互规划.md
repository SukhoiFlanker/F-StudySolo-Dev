### 2.2 权益叠加与计费独立性

- **权益叠加**：用户可同时购买任意包月会员和按量加购包，所有权益**直接叠加**。例如Pro版用户加购+5GB存储后，总存储配额变为3GB+5GB=8GB；加购+5个工作流后，总工作流上限变为50+5=55个。
- **计费周期分别计算**：包月会员和每个加购项拥有独立的计费周期和自动续费设置。系统通过不同的数据库表记录（会员订阅表、加购项表），互不影响。
- **续费/取消独立**：用户可单独取消加购项而不影响主会员，反之亦然。主会员过期时，所有加购项同步失效（按保留期规则处理）；主会员续费后，加购项需重新购买或自动恢复（取决于过期时长）。

### 2.3 会员过期与数据保留机制

| 时间节点 | 状态 | 用户可操作行为 | 系统行为 |
|:---|:---|:---|:---|
| **过期当天（第0天）** | 会员过期 | 可查看/下载/删除已有数据 | 发送第1次提醒邮件+站内通知 |
| | | ❌ 禁止新建工作流 | 新工作流创建按钮禁用 |
| | | ❌ 禁止上传新文件 | 上传功能禁用 |
| **过期后第1-7天** | 宽限期 | 可查看/下载/删除数据 | 每日弹窗提醒“距离数据删除还有X天” |
| | | ✅ 可随时续费恢复所有权益 | 续费后立即恢复全部功能 |
| **过期后第8-14天** | 最后通知期 | 可下载/删除数据 | 第10天发送第2次提醒邮件 |
| | | ❌ 不可上传新内容 | 界面明确显示删除倒计时 |
| **过期后第15天** | **数据清理日** | 所有超量数据进入**不可见状态** | 自动执行清理脚本 |
| | | 可联系客服申请数据恢复（7天内） | 数据标记为“待删除”，但物理删除延迟7天 |
| **过期后第22天** | 永久删除 | 数据不可恢复 | 物理删除数据，释放存储空间 |

> **加购项处理**：主会员过期时，所有加购项同时失效；若主会员在宽限期内续费，加购项需单独重新购买（或系统可设计为自动恢复最后有效的加购配置，但需用户确认续费）。建议设计为：续费主会员后，系统提示“您之前的加购项已过期，如需继续使用请重新购买”，并提供一键复购选项。

### 2.4 按量加购的自动续费逻辑

- 开通加购项时默认开启自动续费，每月扣款日前3天发送提醒。
- 扣款失败有3天宽限期，期间加购权益保留。
- 连续2个月扣款失败，自动取消加购项。
- 加购项取消后，对应的资源配额即时收回（如存储空间减少），但已存储的数据不受影响（仅当总使用量超过新配额时触发只读模式）。
