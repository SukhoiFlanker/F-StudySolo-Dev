## 一、全局 UI/UX 设计理念

1. **响应式断点剧变**：彻底摒弃简单的媒体查询伸缩。桌面端（三分栏）和移动端（上下夹击）采用完全不同的交互模式和组件层级。
2. **呼吸感与微动画**：大面积留白，结合 Framer Motion 实现丝滑过渡；运行中的 AI 节点必须有明确的生命周期表现（Pending、Running 下的呼吸灯、Done）。
3. **组件库底座**：基于 Shadcn/UI + Tailwind CSS v4 构建，采用 OKLCH 色彩空间，严格遵守无障碍（WAI-ARIA）和触摸优先（Touch-first）原则。

## 二、桌面端核心：工作流面板 (Workspace)

桌面端采用具有极强生产力属性的**三栏式弹性布局（Left-Center-Right）**。

### 1. 左侧边栏 (Left Sidebar：工作流调度中枢)

- **尺寸状态**：可展开（~280px）/ 可收起收缩至图标大小（~64px）。
- **展示内容**：平铺展现用户的“工作流 (Workflow)”列表。
- **微交互设计**：
  - **核心操作项**：工作流名称右侧悬浮展现【开始执行 (▶)】和【设置/配置 (⚙)】按钮。
  - **快捷执行**：点击【开始执行】，即使当前在看另外一个工作流的画布，该工作流也会在后台静默启动/执行。
  - **进入画布**：点击工作流卡片主体，右侧画布区域自动平滑切换至该工作流。
  - **右键菜单**：右键点击卡片（Context Menu），可直接呼出配置项（重命名、删除、快速设置）。

### 2. 中间核心区 (Main Canvas：无限画布)

- **基本形态**：基于 `@xyflow/react` 的无限平移拖拽画布。
- **阶段性防干预设计**：
  - **AI 规划期（Analyzer & Planner）**：最初建立工作流时，用户无法点击或参与，画布呈现“加载/骨架屏/闪烁链接”动效。
  - **可参与期（Execution & HITL）**：当节点规划完毕或节点中断时，画布状态解封。用户可双击节点、修改其中的文本内容或 JSON 参数，随后继续流转。

### 3. 右侧边栏 (Right Sidebar：透明化 / 人机协同监控面板)

- **通用入口**：包含用户个人头像、全局设置入口（User Settings）。
- **核心功能 — 线性侧边栏 (Linear Execution View)**：
  - 点击侧边栏展开后，将中间散乱的二维画布网络，**以向下的“线性时间线 (Timeline)”方式展平展示**。
  - **全量 JSON 与输入输出**：展示所有 AI 节点在流转过程中产生的真实 JSON 参数、提词注入情况、中间态交互数据。
  - **调优与透明化**：帮助用户（尤其是极客玩家）完全掌控和调节大模型的对话机制。实现 AI 执行过程 100% 透明化（即所谓“透明化运行看板”）。

## 三、移动端优化：重构交互体验

手机端寸土寸金，强行复用侧边栏会导致灾难级体验。因此对移动端布局进行彻底重构：

### 1. 顶部下拉面板 (Top Expanding Navigation)

- **取代左侧边栏**：移动端不设侧拉抽屉。而是顶部常驻一个紧凑的 NavBar。
- **下拉展开空间**：点击顶部导航后，从上至下平滑展开一个占据 **屏幕约 1/3 (33vh)** 的面板。
- **横向卡片流**：工作流列表以**卡片形式 (Card Carousel)** 横向排列，用户可以通过左右滑动查看不同的工作流。
- **卡片操作**：卡片上整合了【开始执行】与【设置】按钮；支持长按调出配置菜单。

### 2. 中间核心区 (Mobile Canvas)

- **手势优化**：支持双指缩放（Pinch-to-zoom）、单指平移。
- **防误触机制**：节点在移动端采用更激进的吸附范围与防误判逻辑。点击节点自动弹出底部抽屉（Bottom Drawer）以查看和修改该节点内容，而不是在画布上弹窗。

### 3. 底部导航栏 (Bottom Navbar)

- **功能锚定**：由于手机屏幕加长，核心操作必须在“大拇指热区”。屏幕最底端常驻一层毛玻璃 Bottom Nav。
- **入口分布**：
  - `[+] 新建` （居中高亮/浮动FAB）
  - `我的工作流` / `画布看板`
  - `用户设置` / `数据中心`

## 四、透明化线性看板与人机协同交互

为落实“绝不等盲盒”的理念，右侧边栏的线性视图（和移动端的底部抽屉）承担了重要的**人机同步操作**职责：

1. **状态标识**：
   - 灰色：等待执行。
   - 绿色呼吸：执行中，通过 SSE 展示一行行的 Markdown 拼装流。
   - 红色：执行出错，此时暴露 AI 的最后一步 JSON 用于诊断。
2. **人工覆写 (Override)**：
   - 线性面板中，允许用户直接点击某个节点生成的“大纲”或“摘要”，将其变为输入框。
   - 用户修改后点击“继续执行”，引擎自动拉取覆盖后的 JSON 进行下一步请求。

## 五、其他重要页面集

除了核心画布页之外，其他重要页面的 UI 规范方向如下：

### 1. 登录 / 注册页 (Auth)

- **布局**：居中卡片式，左侧（桌面端）为品牌配图/插画或流动的代码动效，右侧为表单。
- **鉴权体验**：极简表单框，引入邮箱验证码发送时的“按钮内联倒计时”。输入采用清晰的 Focus Ring 效验。

### 2. 工作流弹窗配置 / 设置页 (Settings Dialog)

- **布局**：桌面端使用模态弹窗 (Dialog) 覆于画布之上，移动端使用底部全屏升起抽屉 (Drawer)。
- **交互**：无需繁琐的“保存”按钮，重要开关采用“即时生效 (Optimistic UI 结合离线存储)”机制加载，体验更加现代。

### 3. 模板广场 (Template Market)

- **瀑布流 / 响应式网格 (Grid)**。
- 鼠标悬停（Hover）在模板上时，呈现炫酷的该工作流结构的“微缩模型展示”或发光动效。
- 提供【一键克隆】体验，点击即自动弹回 Workspace，工作流克隆即刻完成。
